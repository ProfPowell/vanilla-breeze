font-display Strategies

Different loading behaviors for custom fonts.

font-display: swap

Shows fallback immediately, swaps when loaded. Best for body text. May cause layout shift (CLS).

font-display: block

Brief invisible text period, then shows font. Use for icon fonts or critical branding.

font-display: fallback

Short block period, short swap period. Balanced approach for most use cases.

font-display: optional

Uses font only if already cached. Best for performance-critical pages.

/* Google Fonts with display=swap */
<link href="https://fonts.googleapis.com/css2?family=Inter&display=swap">

/* Font Loading API for more control */
document.fonts.load('1em "Inter"').then(() => {
  document.body.classList.add('fonts-loaded');
});

/* CSS with fallback */
body {
  font-family: "Inter", system-ui, sans-serif;
}

Best Practices

Guidelines for using custom fonts in Vanilla Breeze themes.

/* 1. Preconnect to font origins */
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

/* 2. Limit font weights and styles */
/* Load only what you need: */
family=Inter:wght@400;600  /* Not wght@100..900 */

/* 3. Use fallback stacks */
--font-display: "Playfair Display", Georgia, "Times New Roman", serif;

/* 4. Subset for specific characters if needed */
<link href="...&text=ABCDEFG">

/* 5. Consider variable fonts for flexibility */
family=Inter:wght@400..700  /* Range instead of discrete weights */
