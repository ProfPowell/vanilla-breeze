Data (<data>)

The data element links human-readable content with a machine-readable value, enabling content to be processed by scripts while remaining readable to users.

Description

The <data> element associates a piece of content with a machine-readable translation via its value attribute. This allows you to display human-friendly text while providing structured data for scripts, search engines, or other processing.

Semantic Meaning

The data element provides a bridge between human-readable content and machine-readable values:

  • The visible text content is what users see and read
  • The value attribute contains the machine-processable form
  • Scripts can access the value property via JavaScript
  • Useful for microdata and structured data applications

When to Use

  • Product IDs: Display SKU while storing internal ID
  • Prices: Show formatted price, store numeric value
  • Quantities: Display "Five items" but store 5
  • Status codes: Show "Completed" while storing status code
  • Any dual-format data: When you need both human and machine formats

When NOT to Use

  • Dates/times: Use <time> instead
  • Pure display: If no machine processing is needed, plain text suffices

Variants

Default

Displays in monospace font at slightly smaller size, indicating machine-readable content.

<data value="12345">12345</data>

JavaScript Access

The value attribute can be accessed via the value property in JavaScript.

// Get the machine-readable value
const dataElement = document.querySelector('data');
const machineValue = dataElement.value; // "12345"

// Or using dataset for custom attributes
const displayText = dataElement.textContent; // "Product 12345"

Accessibility

  • Screen readers: Read the visible text content, not the value attribute
  • No special announcement: The data element doesn't convey special meaning to assistive technologies
  • Ensure visible content is meaningful: Don't rely solely on the value attribute for important information

Related Elements

  • <time> - Specifically for dates and times (use instead of data for temporal values)
  • <output> - For calculation results in forms
  • <code> - For code snippets (different semantic purpose)
