heading-links

Add anchor links to headings for easy navigation and URL sharing.

Overview

The <heading-links> component wraps content and automatically adds clickable anchor links to headings. Clicking a link updates the URL hash, scrolls to the heading, and copies the link to the clipboard.

This component is used throughout this documentation site. Hover over any heading on this page to see the link icon appear.

<heading-links> <h2 id="usage">Usage</h2> <p>Content here...</p> <h3>Example</h3> <p>More content...</p> </heading-links>

Attributes

Attribute Type Default Description
data-levels string "h2,h3" Comma-separated list of heading levels to process.

How It Works

When the component connects:

  1. Scans for headings matching the specified levels within its content.
  2. Generates IDs for headings that don't have one (based on heading text).
  3. Adds anchor links with a link icon that appear on hover.
  4. Watches for changes using MutationObserver to handle dynamic content.

On Click

When a heading link is clicked:

  1. Updates the URL hash
  2. Smoothly scrolls to the heading
  3. Focuses the heading for accessibility
  4. Copies the full URL to clipboard
  5. Shows brief visual feedback (checkmark icon)
  6. Announces "Link copied" to screen readers

Heading Levels

Use data-levels to control which headings get anchor links.

<!-- Default: h2 and h3 --> <heading-links>...</heading-links> <!-- All headings h2 through h4 --> <heading-links data-levels="h2,h3,h4">...</heading-links> <!-- Only h2 headings --> <heading-links data-levels="h2">...</heading-links>

Auto-Generated IDs

Headings without an id attribute get one automatically generated from their text content:

<!-- Input --> <h2>Getting Started</h2> <!-- Becomes --> <h2 id="getting-started">Getting Started</h2>

The ID generation:

  • Converts to lowercase
  • Replaces spaces with hyphens
  • Removes special characters
  • Limits length to 50 characters

For best results, provide explicit IDs for important headings to ensure stable URLs.

Events

The component dispatches events when a heading link is activated.

Event Detail Description
heading-navigate { id, url } Fired when a heading link is clicked.
const headingLinks = document.querySelector('heading-links'); headingLinks.addEventListener('heading-navigate', (e) => { console.log('Navigated to:', e.detail.id); console.log('URL:', e.detail.url); });

Keyboard Support

Key Action
Tab Focus the anchor link
Enter Activate the link (navigate and copy)

Accessibility

ARIA Attributes

  • Each anchor link has aria-label describing its purpose (e.g., "Link to Usage")
  • Headings get tabindex="-1" to allow focus after navigation
  • Clipboard success is announced via role="status" live region

Screen Reader Experience

When navigating to a heading via the link:

  • The heading receives focus and is announced
  • "Link copied to clipboard" is announced as a status message

Best Practices

  • Use semantic heading hierarchy (h2 before h3, etc.)
  • Provide meaningful heading text
  • Consider providing explicit IDs for important sections

Styling

The anchor links can be styled using CSS.

/* The anchor link element */ heading-links .heading-anchor { display: inline-flex; align-items: center; margin-inline-start: var(--size-xs); padding: var(--size-3xs); color: var(--color-text-muted); opacity: 0; transition: opacity var(--duration-fast) var(--ease-default); } /* Show on heading hover */ heading-links :is(h1, h2, h3, h4, h5, h6):hover .heading-anchor { opacity: 1; } /* Hover state */ heading-links .heading-anchor:hover { color: var(--color-interactive); background: var(--color-surface-raised); } /* Success state after copy */ heading-links .heading-anchor.copied { color: var(--color-success); }

With page-toc

The heading-links component works seamlessly with <page-toc>. The page-toc component can generate a table of contents from headings that have been enhanced with IDs by heading-links.

<layout-sidebar data-side="end"> <aside> <page-toc data-levels="h2,h3" data-scope="article"></page-toc> </aside> <article> <heading-links data-levels="h2,h3"> <h2>Section One</h2> <p>Content...</p> <h3>Subsection</h3> <p>More content...</p> </heading-links> </article> </layout-sidebar>

Dynamic Content

The component uses MutationObserver to watch for new headings added to the DOM. New headings are automatically enhanced.

const container = document.querySelector('heading-links'); const newSection = document.createElement('section'); newSection.innerHTML = '<h2>New Section</h2><p>Content</p>'; container.appendChild(newSection); // The h2 is automatically enhanced with an anchor link

Use Cases

Documentation Sites

Enable deep linking to specific sections:

<article> <heading-links data-levels="h2,h3,h4"> <h2 id="installation">Installation</h2> <h3 id="npm">npm</h3> <h3 id="cdn">CDN</h3> <h2 id="usage">Usage</h2> </heading-links> </article>

Blog Posts

Allow readers to share specific sections:

<article class="blog-post"> <heading-links data-levels="h2"> <h2>Introduction</h2> <p>...</p> <h2>Key Findings</h2> <p>...</p> <h2>Conclusion</h2> <p>...</p> </heading-links> </article>

FAQ Pages

Link directly to specific questions:

<heading-links data-levels="h3"> <h2>Frequently Asked Questions</h2> <h3 id="faq-pricing">How much does it cost?</h3> <p>...</p> <h3 id="faq-support">How do I get support?</h3> <p>...</p> </heading-links>
