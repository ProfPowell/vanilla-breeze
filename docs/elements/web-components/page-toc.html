page-toc

Table of contents with scroll-spy highlighting for documentation and long-form content.

Overview

The <page-toc> component generates a table of contents from page headings or enhances existing markup with scroll-spy functionality. As users scroll, the current section is highlighted in the ToC.

Look to the right side of this page to see it in action. The "On this page" panel highlights the current section as you scroll.

<!-- Auto-generated ToC --> <page-toc data-levels="h2,h3" data-scope="main"></page-toc>

Attributes

Attribute Type Default Description
data-levels string "h2,h3" Comma-separated list of heading levels to include.
data-scope string "main" CSS selector for the content scope to scan for headings.
data-title string "On this page" Title shown at the top of the ToC.

Two Modes

The component supports two usage modes:

Auto-Generated Mode

When the component is empty, it automatically scans the page for headings and builds the ToC:

<page-toc data-levels="h2,h3" data-scope="article" data-title="Contents"> </page-toc>

Manual Markup Mode

Provide your own markup for progressive enhancement. The component adds scroll-spy to existing links:

<page-toc> <details class="details" open> <summary class="summary">On this page</summary> <nav class="nav" aria-label="On this page"> <ul class="list"> <li class="item"><a href="#overview" class="link">Overview</a></li> <li class="item"><a href="#usage" class="link">Usage</a></li> <li class="item"><a href="#api" class="link">API</a></li> </ul> </nav> </details> </page-toc>

Manual mode is useful when you need custom ordering or want to include only specific headings.

How It Works

Initialization

  1. Detects mode (auto vs manual) by checking for existing markup
  2. In auto mode, scans for headings within the specified scope
  3. Builds or enhances the ToC structure
  4. Sets up scroll-spy using IntersectionObserver

Scroll-Spy

The component uses IntersectionObserver to track which headings are visible:

  • Observes all headings in the ToC
  • Highlights the first visible heading
  • Updates as user scrolls
  • Syncs with URL hash changes

Responsive Behavior

  • Wide screens (1024px+): ToC is always visible, sticky positioned
  • Narrow screens: ToC is collapsible via details/summary

Customizing Levels

Use data-levels to control which headings appear in the ToC:

<!-- Just h2 headings --> <page-toc data-levels="h2"></page-toc> <!-- h2, h3, and h4 --> <page-toc data-levels="h2,h3,h4"></page-toc> <!-- Only h3 within a specific section --> <page-toc data-levels="h3" data-scope=".api-docs"></page-toc>

Customizing Scope

Use data-scope to limit which part of the page is scanned for headings:

<!-- Default: scan main element --> <page-toc data-scope="main"></page-toc> <!-- Scan a specific article --> <page-toc data-scope="article"></page-toc> <!-- Scan by class --> <page-toc data-scope=".content-area"></page-toc> <!-- Scan by ID --> <page-toc data-scope="#main-content"></page-toc>

Events

The component dispatches events when navigation occurs.

Event Detail Description
toc-navigate { id } Fired when a ToC link is clicked.
const toc = document.querySelector('page-toc'); toc.addEventListener('toc-navigate', (e) => { console.log('Navigated to section:', e.detail.id); });

JavaScript API

Method Description
refresh() Rebuild the ToC (useful after dynamic content changes).
const toc = document.querySelector('page-toc'); // After adding new content toc.refresh();

Styling

The component uses scoped class names that can be styled with CSS:

@scope (page-toc) { /* Collapsible container */ .details { font-size: var(--font-size-sm); } /* Toggle header */ .summary { font-weight: 600; cursor: pointer; } /* Navigation container */ .nav { padding: var(--size-s); } /* List of links */ .list { list-style: none; margin: 0; padding: 0; } /* Individual list items */ .item { margin: 0; } /* Indent nested levels */ .item[data-level="1"] { padding-inline-start: var(--size-m); } /* ToC links */ .link { display: block; padding: var(--size-2xs) var(--size-xs); color: var(--color-text-muted); text-decoration: none; border-inline-start: 2px solid transparent; } /* Active link (current section) */ .link.active { color: var(--color-interactive); border-inline-start-color: var(--color-interactive); font-weight: 500; } }

Layout Patterns

Common layouts for positioning the ToC:

Sidebar Layout

Place the ToC in a sidebar using layout-sidebar:

<layout-sidebar data-side="end" data-sidebar-width="narrow"> <aside> <page-toc data-levels="h2,h3" data-scope="article"></page-toc> </aside> <article> <heading-links> <h2 id="intro">Introduction</h2> <p>...</p> <h2 id="features">Features</h2> <p>...</p> </heading-links> </article> </layout-sidebar>

Sticky Positioning

The component includes sticky positioning styles for wide screens:

@media (min-width: 1024px) { page-toc { position: sticky; top: var(--size-xl); } }

With heading-links

The page-toc component works seamlessly with <heading-links>. The page-toc waits for heading-links to add IDs before scanning:

<layout-sidebar data-side="end"> <aside> <page-toc data-levels="h2,h3" data-scope="article"></page-toc> </aside> <article> <heading-links data-levels="h2,h3"> <!-- Headings here will get IDs from heading-links --> <!-- page-toc will pick them up for the ToC --> <h2>Section One</h2> <p>Content...</p> </heading-links> </article> </layout-sidebar>

Accessibility

ARIA Attributes

  • The nav element has aria-label matching the ToC title
  • Active links have aria-current="true"

Keyboard Navigation

  • All ToC links are keyboard accessible
  • On narrow screens, the details/summary provides keyboard-accessible expand/collapse

Focus Management

When clicking a ToC link:

  • The page smoothly scrolls to the heading
  • Focus moves to the heading for screen reader announcement

Use Cases

Documentation

<page-toc data-levels="h2,h3,h4" data-scope="article" data-title="API Reference"> </page-toc>

Blog Posts

<page-toc data-levels="h2" data-scope=".blog-content" data-title="Jump to"> </page-toc>

Long Forms

<page-toc data-levels="h2" data-scope="form" data-title="Sections"> </page-toc>

Limitations

  • Headings must have IDs for the ToC to link to them (use heading-links to auto-generate)
  • In auto mode, only headings within the specified scope are included
  • Dynamic content changes require calling refresh() to update the ToC
