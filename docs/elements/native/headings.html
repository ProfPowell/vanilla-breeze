Headings (h1-h6)

Headings create a hierarchical structure for your content, helping users and assistive technologies navigate and understand the document outline.

  • Headings
  • Paragraph
  • Blockquote
  • Code
  • Horizontal Rule

Overview

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). Headings are essential for:

  • Creating a logical document outline
  • Enabling screen reader users to navigate by heading
  • Improving SEO by signaling content structure to search engines
  • Providing visual hierarchy to guide readers through content

Heading Level 1 (h1)

The <h1> element represents the main heading of a page or major section. There should typically be only one <h1> per page.

Main Page Title

When to Use

  • As the primary title of your page
  • To identify the main topic or purpose of the page
  • Usually appears at the top of the main content area

Heading Level 2 (h2)

The <h2> element represents major sections within the page. It directly supports the main heading.

Major Section Title

Content for this section follows the h2 heading.

When to Use

  • For top-level content sections
  • To divide the page into logical parts
  • As section titles in long-form content

Heading Level 3 (h3)

The <h3> element represents subsections within an <h2> section.

Subsection Title

Content for this subsection follows the h3 heading.

When to Use

  • To divide major sections into smaller topics
  • For grouping related content within a section

Heading Levels 4-6 (h4, h5, h6)

Lower heading levels provide additional granularity for complex documents. Use sparingly as deeply nested headings can indicate content organization issues.

CSS Properties

Headings in Vanilla Breeze use the following shared styles:

h1, h2, h3, h4, h5, h6 { font-weight: 600; line-height: var(--line-height-tight); text-wrap: balance; } h1 { font-size: var(--font-size-4xl); } h2 { font-size: var(--font-size-3xl); } h3 { font-size: var(--font-size-2xl); } h4 { font-size: var(--font-size-xl); } h5 { font-size: var(--font-size-lg); } h6 { font-size: var(--font-size-md); font-weight: 700; }

Key Features

  • text-wrap: balance - Distributes text evenly across lines to avoid orphaned words
  • line-height: tight - Reduced line height appropriate for short heading text
  • font-weight: 600 - Semi-bold weight for emphasis (700 for h6)

Accessibility

Heading Structure Rules

  • Don't skip levels - Go from h1 to h2 to h3, not h1 to h3
  • One h1 per page - Use a single h1 for the main page title
  • Logical nesting - Lower levels should be nested within higher levels
  • Content, not style - Choose heading level based on structure, not appearance

Screen Reader Navigation

Screen reader users often navigate pages by jumping between headings. A well-structured heading hierarchy allows them to:

  • Quickly scan the page content
  • Jump directly to relevant sections
  • Understand the relationship between content sections

Example: Good vs Bad Structure

Good Structure: <h1>Page Title</h1> <h2>Section A</h2> <h3>Subsection A.1</h3> <h3>Subsection A.2</h3> <h2>Section B</h2> <h3>Subsection B.1</h3>
Bad Structure (skips levels): <h1>Page Title</h1> <h3>Section A</h3> <!-- Skipped h2! --> <h4>Subsection</h4> <h2>Section B</h2> <!-- Back to h2? -->

Related Elements

  • <p> - Paragraph text that follows headings
  • <section> - Semantic sectioning element often used with headings
  • <article> - Self-contained content with its own heading hierarchy
  • <hr> - Thematic break between sections
