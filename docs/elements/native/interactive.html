Interactive Elements

Native interactive elements: details/summary, dialog, menu, and search. No JavaScript required for basic functionality.

Details / Summary

Native disclosure widget for expandable content. No JavaScript required.

Single Details

Click to expand

This content is revealed when the details element is opened. The summary acts as a clickable header that toggles the visibility of the content inside.

<details> <summary>Click to expand</summary> <p>Hidden content revealed when opened.</p> </details>

Open by Default

Use the open attribute to show content initially.

Already expanded

This details element starts in the open state. Users can click to collapse it.

<details open> <summary>Already expanded</summary> <p>Starts in the open state.</p> </details>

Accordion Pattern

Adjacent details elements stack with connected borders.

What is Vanilla Breeze?

Vanilla Breeze is a layered HTML component system that extends HTML's native model with CSS custom properties, custom elements, and web components.

How does progressive enhancement work?

Components are built to work without JavaScript first. CSS provides visual styling and layout. JavaScript then enhances with interactivity and accessibility features.

What browsers are supported?

All modern browsers are supported. The library uses native CSS features like @layer, custom properties, and :has() which have broad support.

<details> <summary>First item</summary> <p>Content for first item.</p> </details> <details> <summary>Second item</summary> <p>Content for second item.</p> </details>

Exclusive Accordion (name attribute)

Use the name attribute for exclusive open behavior (only one open at a time).

First Question

Only one panel can be open at a time when details elements share the same name attribute.

Second Question

Opening this will automatically close the other panels.

Third Question

This is a native browser feature requiring no JavaScript.

<details name="faq" open> <summary>First Question</summary> <p>Only one open at a time.</p> </details> <details name="faq"> <summary>Second Question</summary> <p>Opens and closes others.</p> </details>

Tooltip

Native tooltips using the Popover API and CSS Anchor Positioning. Minimal JavaScript for hover events.

Basic Tooltip

Connect a trigger to a tooltip with aria-describedby. The tooltip uses popover="hint" for top-layer rendering.

This is a tooltip!
<button aria-describedby="tip1">Hover me</button> <div id="tip1" popover="hint" role="tooltip"> This is a tooltip! </div>

Position Variants

Use data-position to control tooltip placement. CSS Anchor Positioning handles the positioning automatically.

Above the trigger
Below the trigger
Left of trigger
Right of trigger
<button aria-describedby="tip-bottom">Bottom</button> <div id="tip-bottom" popover="hint" role="tooltip" data-position="bottom"> Below the trigger </div>

Click-Triggered (No JS)

Use popovertarget for click-triggered tooltips. No JavaScript required.

Click tooltip - click outside to dismiss
<button popovertarget="tip-click">Click me</button> <div id="tip-click" popover="auto" role="tooltip"> Click to dismiss or click outside </div>

With Arrow

Add a .tooltip-arrow element for a pointing arrow.

Tooltip with arrow
<button aria-describedby="tip-arrow">With arrow</button> <div id="tip-arrow" popover="hint" role="tooltip"> Tooltip with arrow <span class="tooltip-arrow" aria-hidden="true"></span> </div>

Initialization

Import tooltip-init.js to enable hover/focus behavior for native tooltips.

// Auto-init on import import '/src/utils/tooltip-init.js'; // Or manual init for dynamic content import { initTooltips } from '/src/utils/tooltip-init.js'; initTooltips(document.querySelector('.dynamic-content'));

Browser Support

Uses CSS Anchor Positioning in supporting browsers with JavaScript fallback positioning for others.

Feature Support
Popover API Chrome 114+, Safari 17+, Firefox 125+
CSS Anchor Positioning Chrome 125+, Safari 18+

Native vs tooltip-wc

Choose based on your needs:

Use Case Pattern
Simple hover tooltips Native (this pattern)
Generated from template tooltip-wc
Minimal markup tooltip-wc
Click-triggered (no JS) Native (this pattern)

Dialog

Native modal dialog with backdrop, animation, and focus management.

Basic Dialog

Open dialogs declaratively with the Invokers API (commandfor + command="show-modal"). No JavaScript required.

Basic Dialog

This is a native HTML dialog element. It has a backdrop and can be closed with ESC.
<button type="button" commandfor="my-dialog" command="show-modal"> Open Dialog </button> <dialog id="my-dialog"> <header> <h3>Dialog Title</h3> </header> <p>Dialog content here.</p> <footer> <button type="button" commandfor="my-dialog" command="close">Close</button> </footer> </dialog>

Dialog with Form

Subscribe to Newsletter
<button commandfor="form-dialog" command="show-modal">Open</button> <dialog id="form-dialog"> <header> <h3>Subscribe</h3> </header> <form method="dialog"> <label for="email">Email</label> <input type="email" id="email" /> <footer> <button type="button" class="secondary" commandfor="form-dialog" command="close">Cancel</button> <button type="submit">Subscribe</button> </footer> </form> </dialog>

Confirmation Dialog

Confirm Deletion

Are you sure you want to delete this item? This action cannot be undone.
<button commandfor="confirm-dialog" command="show-modal">Delete Item</button> <dialog id="confirm-dialog"> <header> <h3>Confirm Deletion</h3> </header> <p>Are you sure? This cannot be undone.</p> <footer> <button type="button" class="secondary" commandfor="confirm-dialog" command="close">Cancel</button> <button type="button" commandfor="confirm-dialog" command="close">Delete</button> </footer> </dialog>

Dialog Anatomy

Dialogs support header, section (or direct content), and footer for structured layouts.

Dialog Header

The dialog body contains the main content. It can include any HTML elements.

Long content will scroll within the dialog while header and footer remain fixed.
<button commandfor="my-dialog" command="show-modal">Open</button> <dialog id="my-dialog"> <header> <h3>Dialog Header</h3> </header> <section> <p>Main dialog content.</p> </section> <footer> <button type="button" class="secondary" commandfor="my-dialog" command="close">Cancel</button> <button type="button" commandfor="my-dialog" command="close">Confirm</button> </footer> </dialog>

Menu

Menu patterns for toolbars, context menus, and button groups.

.toolbar

Horizontal toolbar with icon or text buttons.

    •
    <menu class="toolbar"> <li><button type="button" class="ghost">Bold</button></li> <li><button type="button" class="ghost">Italic</button></li> <li><button type="button" class="ghost">Underline</button></li> </menu>

    .context

    Dropdown-style context menu with separators.

    •
    <menu class="context"> <li><button type="button">Cut</button></li> <li><button type="button">Copy</button></li> <li><button type="button">Paste</button></li> <li role="separator"></li> <li><button type="button">Select All</button></li> </menu>

    .context with Links

  • View Profile
  • Settings
  • Help
  • Sign Out
    •
    <menu class="context"> <li><a href="#">View Profile</a></li> <li><a href="#">Settings</a></li> <li role="separator"></li> <li><a href="#">Sign Out</a></li> </menu>

    .vertical

    Vertical menu layout.

    •
    <menu class="vertical"> <li><button type="button" class="ghost">Dashboard</button></li> <li><button type="button" class="ghost">Projects</button></li> <li><button type="button" class="ghost">Settings</button></li> </menu>

    .icons

    Icon-only toolbar.

    •
    <menu class="toolbar icons"> <li> <button type="button" class="ghost" aria-label="Bold"> <svg>...</svg> </button> </li> <li> <button type="button" class="ghost" aria-label="Italic"> <svg>...</svg> </button> </li> </menu>

    .pills

    Pill-shaped menu items.

    •
    <menu class="toolbar pills"> <li><button type="button" class="ghost">All</button></li> <li><button type="button">Active</button></li> <li><button type="button" class="ghost">Completed</button></li> </menu>

    .compact

    Reduced spacing for dense interfaces.

    •
    <menu class="context compact"> <li><button type="button">Item 1</button></li> <li><button type="button">Item 2</button></li> <li><button type="button">Item 3</button></li> </menu>

    Search

    Semantic search landmark with styling variants.

    Default

    <search> <input type="search" placeholder="Search..." /> </search>

    .inline

    Search input with button.

    <search class="inline"> <input type="search" placeholder="Search..." /> <button type="submit">Search</button> </search>

    .compact

    Smaller search for headers or sidebars.

    <search class="compact"> <input type="search" placeholder="Quick search..." /> <button type="submit">Find</button> </search>

    .rounded

    Pill-shaped search input.

    <search class="inline rounded"> <input type="search" placeholder="Search..." /> <button type="submit">Go</button> </search>

    .expanded

    Full-width search.

    <search class="expanded"> <input type="search" placeholder="Search the entire site..." /> </search>

    .header

    Constrained width for header placement.

    Site Logo Nav Items
    <search class="header inline"> <input type="search" placeholder="Search..." /> <button type="submit" class="small">Go</button> </search>

    With Form

    Search can wrap a form for enhanced functionality.
    <search> <form action="/search" method="get" class="inline"> <label for="search-q" class="visually-hidden">Search</label> <input type="search" id="search-q" name="q" placeholder="Search..." /> <button type="submit">Search</button> </form> </search>
