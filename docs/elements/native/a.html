Anchor (<a>)

The anchor element creates hyperlinks to web pages, files, email addresses, locations within the same page, or any other URL.

Description

The <a> element (anchor) is the fundamental building block of the web, connecting pages and resources through hyperlinks. It can link to external websites, internal pages, sections within a page (via fragment identifiers), email addresses, phone numbers, and downloadable files.

Semantic Meaning

The anchor element represents a hyperlink. When it has an href attribute, it indicates a link to another resource. Without href, it represents a placeholder for a link that might otherwise have been placed there.

  • Screen readers announce links distinctly from other content
  • Search engines use links to understand page relationships and site structure
  • Users can navigate links using keyboard (Tab key)

When to Use

  • Navigation: Use for links in site navigation, menus, and breadcrumbs
  • In-page links: Use for table of contents linking to sections
  • External resources: Use for linking to external sites and references
  • Downloads: Use with download attribute for downloadable files
  • Contact links: Use mailto: and tel: protocols

When NOT to Use

  • Buttons: Use <button> for actions that don't navigate
  • Form submission: Use <button type="submit"> instead
  • JavaScript actions: Use buttons for toggles, modals, or state changes

Variants

Default Link

Standard link with underline and interactive color.

This is a default link in a paragraph.

<a href="#">default link</a>

.muted

Less prominent link using muted text color. Good for footer links or secondary navigation.

This is a muted link that is less prominent.

<a href="#" class="muted">muted link</a>

.plain

No underline by default, reveals underline on hover.

This is a plain link without default underline.

<a href="#" class="plain">plain link</a>

.external

Displays an arrow indicator for external links.

Visit Example.com for more information.

<a href="https://example.com" class="external">Example.com</a>

Links Wrapping Content

Links automatically remove underline when wrapping images or cards.

Key Attributes

Attribute Description
href The URL the link points to (required for functional link)
target Where to open the link (_blank for new tab)
rel Relationship between pages (e.g., noopener, noreferrer)
download Prompts download instead of navigation
hreflang Language of the linked document

CSS Properties

Property Value
color var(--color-interactive)
text-decoration underline
text-decoration-thickness 1px (2px on hover)
text-underline-offset 0.15em

Accessibility

  • Link text should be descriptive: Avoid "click here" or "read more" - use meaningful text that describes the destination
  • Color alone is not sufficient: Links have underlines by default to distinguish them from regular text
  • Focus states: Links have visible focus rings for keyboard navigation
  • External links: Consider informing users when links open in new tabs
  • Skip links: Use anchor links to create "skip to content" navigation
