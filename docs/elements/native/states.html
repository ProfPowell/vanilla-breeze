States

State attributes that apply loading, disabled, and other visual states to any element.

data-loading

Apply data-loading to any element to show a loading state with animated shimmer effect. Works on native elements, custom elements, and layout primitives.

Attributes

Attribute Values Description
data-loading (presence) Default loading state - dims content with shimmer overlay
data-loading="hide" hide Hides content completely (visibility: hidden)
data-loading="minimal" minimal Subtle pulse effect without sweep animation
data-loading="skeleton" skeleton Shows content shapes - each child becomes a gray placeholder

Basic Usage

Add data-loading to any container to show its loading state.

Card Title

This card is in a loading state. Content is dimmed with a shimmer overlay.

Card Title

This card is loaded. Normal appearance without loading state.
<layout-card data-loading> <h3>Card Title</h3> <p>Content here...</p> </layout-card>

Works on Any Element

The data-loading attribute works on native HTML elements, custom elements, and layout primitives.

Native Elements

<figure data-loading> <img src="photo.jpg" alt="..." /> <figcaption>Caption</figcaption> </figure> <button data-loading>Submit</button>

Variants

Hide Content

Use data-loading="hide" to completely hide content instead of dimming it.

Default

Content is dimmed but visible

Hide Variant

Content is completely hidden
<layout-card data-loading="hide"> <!-- Content hidden until loaded --> </layout-card>

Minimal Animation

Use data-loading="minimal" for a subtle pulse without the sweep effect.

Minimal Loading

Subtle pulse animation without the sweep overlay.
<layout-card data-loading="minimal"> <!-- Subtle pulse effect --> </layout-card>

Skeleton (Content Shapes)

Use data-loading="skeleton" to show the shape of content. Each child element becomes a gray placeholder, with headings darker than body text to preserve visual hierarchy.

This paragraph text becomes a gray bar showing the approximate shape of the content.

Dynamic Loading States

Toggle the loading state with JavaScript by adding or removing the attribute.

// Start loading card.dataset.loading = ''; // Start loading with variant card.dataset.loading = 'hide'; // Stop loading delete card.dataset.loading;

Accessibility

  • The shimmer animation respects prefers-reduced-motion. Users who prefer reduced motion see a static overlay.
  • Elements with data-loading have pointer-events: none to prevent interaction.
  • Content remains in the DOM for screen readers. Consider adding aria-busy="true" for dynamic loading.
 <layout-card data-loading aria-busy="true"> <!-- Content loading... --> </layout-card>

Choosing a Variant

Each variant serves a different purpose:

  • Default - Overlay shimmer. Best when refreshing existing data.
  • hide - Hides content. Use when content shouldn't be visible during load.
  • minimal - Subtle pulse. Less distracting for quick operations.
  • skeleton - Shows content shapes. Best for initial page load to indicate structure.

All variants keep content in the DOM, eliminating layout shift when loading completes.
