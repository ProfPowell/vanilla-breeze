Card List Component

Template-based list rendering with safe data binding. Only property paths allowed - no expressions.

Inline Data with data-items

Nested Properties

Access nested data with dot notation like user.name or address.city.

Attribute Binding with data-field-attr

Set element attributes using data-field-attr="href: url, title: name" syntax.

Conditional Rendering

Use data-field-if to show elements when a property is truthy, or data-field-unless to hide them.

Security: Unsafe Paths Rejected

Only simple property paths are allowed. Expressions, method calls, and template literals are blocked with console warnings.

Allowed:

name - Simple property
user.email - Nested property
items[0].title - Array index access

Rejected (console warning):

price.toFixed(2) - Method call
`$${price}` - Template literal
stock > 0 ? 'yes' : 'no' - Ternary
price * 1.1 - Arithmetic
