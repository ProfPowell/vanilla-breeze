Subgrid Demo: Definition Lists

Compare traditional dl grid layouts vs subgrid when using HTML5 div wrappers for grouping.

The Problem

HTML5 allows <div> wrappers inside <dl> to group <dt>/<dd> pairs. However, this breaks traditional grid alignment since each wrapper becomes its own grid context.

The CSS

Before (Each Wrapper Independent)

dl > div {
  display: grid;
  grid-template-columns: max-content 1fr; /* Each div sizes independently */
}

After (Subgrid)

dl {
  display: grid;
  grid-template-columns: max-content 1fr; /* Parent defines columns */
}

dl > div {
  display: grid;
  grid-column: span 2;
  grid-template-columns: subgrid; /* Inherit parent columns */
}

Key Difference

  • Before: Each div wrapper has its own max-content calculation
  • After: Parent dl defines columns, wrappers use subgrid to participate in parent's column sizing
  • Result: dt column auto-sizes to the longest term across ALL groups

Complexity Comparison

Metric Before After
Lines of CSS ~4 ~8
Cross-group alignment No Yes
Requires HTML change No No
Works without wrappers Yes Yes (same)

Evaluation

Benefit: Subgrid solves a real problem when div wrappers are needed (for styling borders, hover states, etc.).

Cost: Minimal extra CSS. Only needed when using div wrappers.

Recommendation: Marginal benefit - document as a technique. Most definition lists don't need wrappers, and when they don't, basic grid works fine. Add to documentation rather than core utilities.
