Layout Attributes

Use data-layout on semantic HTML elements for all VB layouts. No wrapper elements needed.

Component Layouts

These layouts work on any element, ideal for cards, sections, and nested content.

Stack Layout

Vertical rhythm with consistent spacing between child elements.

Item 1

Item 2

Item 3

<article data-layout="stack" data-gap="m"> <p>Item 1</p> <p>Item 2</p> <p>Item 3</p> </article>

Cluster Layout

Horizontal grouping with wrapping. Perfect for tags, buttons, and navigation.

HTML CSS JavaScript Web Components
<nav data-layout="cluster" data-gap="s"> <span class="tag">HTML</span> <span class="tag">CSS</span> <span class="tag">JavaScript</span> <span class="tag">Web Components</span> </nav>

Cluster with justify-between

Site Name
About Services Contact
<header data-layout="cluster" data-justify="between"> <strong>Site Name</strong> <nav data-layout="cluster" data-gap="m"> <a href="#">About</a> <a href="#">Services</a> <a href="#">Contact</a> </nav> </header>

Grid Layout

Responsive auto-fit grid. Resize the browser to see columns adapt.

  • Card 1
  • Card 2
  • Card 3
  • Card 4
<ul data-layout="grid" data-min="10rem" data-gap="m"> <li class="card">Card 1</li> <li class="card">Card 2</li> <li class="card">Card 3</li> <li class="card">Card 4</li> </ul>

Center Layout

Max-width container with auto margins. Great for readable content.

This content is centered with a max-width. It won't stretch beyond the specified width.

<main data-layout="center" data-max="narrow"> <p>Centered content...</p> </main>

Sidebar Layout

Two-column flexbox that wraps responsively. Try resizing.

Sidebar
Main content that takes remaining space
<section data-layout="sidebar" data-gap="m"> <nav>Sidebar</nav> <main>Main content...</main> </section>

Split Layout

Two-column ratio grid. Stacks on mobile.

Main (2fr)
Sidebar (1fr)
<section data-layout="split" data-ratio="2:1" data-gap="l"> <article>Main content (2fr)</article> <aside>Sidebar (1fr)</aside> </section>

Switcher Layout

Flexbox that switches from horizontal to vertical based on container width.

Item 1
Item 2
Item 3
<section data-layout="switcher" data-threshold="30rem"> <div>Item 1</div> <div>Item 2</div> <div>Item 3</div> </section> <!-- Stacks when container < 30rem -->

Page Layouts

Full-page layouts using data-layout or data-page-layout on <body>. Semantic elements are automatically placed in grid areas.

Page Stack

Header/Main/Footer with sticky footer. Main expands to fill available space.

<body data-layout="page-stack"> <header>Header</header> <main>Main Content</main> <footer>Footer</footer> </body>

Holy Grail Layout

Classic 3-column layout with header/footer. Uses CSS Grid named areas targeting semantic elements.

<body data-layout="holy-grail"> <header>Header</header> <nav>Nav</nav> <main>Main Content</main> <aside>Aside</aside> <footer>Footer</footer> </body>

Holy Grail (no aside)

The layout automatically adapts when semantic elements are missing.

<body data-layout="holy-grail"> <header>Header</header> <nav>Nav</nav> <main>Main Content</main> <!-- No aside - layout adapts automatically --> <footer>Footer</footer> </body>

Dashboard Layout

App shell with sticky header and sidebar.

<body data-layout="dashboard"> <header>Dashboard Header</header> <nav>Navigation</nav> <main>Dashboard Content</main> </body>

Sidebar Left (Page Layout)

Two-column page layout with navigation on the left.

<body data-page-layout="sidebar-left"> <header>Header</header> <nav> <a href="#">Home</a> <a href="#">About</a> </nav> <main>Main Content</main> <footer>Footer</footer> </body>

Sidebar Right (Page Layout)

Two-column page layout with aside on the right.

<body data-page-layout="sidebar-right"> <header>Header</header> <main>Main Content</main> <aside>Sidebar</aside> <footer>Footer</footer> </body>

Cover Layout

Full-height container with vertically centered principal element.

<body data-layout="cover" data-center> <header>Top content</header> <div data-principal> <h1>Hero Headline</h1> <p>Vertically centered content</p> </div> <footer>Bottom content</footer> </body>
