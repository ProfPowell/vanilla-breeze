Subgrid Demo: Article Layout

Compare independent article/aside layouts vs subgrid for shared vertical rhythm.

The Problem

When an article and aside sit side-by-side, their internal sections don't share a baseline grid. The article's header and aside's header may not align, and content sections have independent heights.

The CSS

Before (Independent)

main {
  display: grid;
  grid-template-columns: 1fr 250px;
  gap: var(--size-l);
}

/* Article and aside manage their own layouts independently */

After (Subgrid)

main {
  display: grid;
  grid-template-columns: 1fr 250px;
  grid-template-rows: auto auto auto auto; /* Define shared row tracks */
  gap: var(--size-l);
}

article {
  display: grid;
  grid-row: span 4;            /* Span 4 rows */
  grid-template-rows: subgrid; /* Inherit parent's row tracks */
}

aside {
  display: grid;
  grid-row: span 4;
  grid-template-rows: subgrid;
}

Key Difference

  • Before: Article and aside have independent vertical flow
  • After: Both participate in parent's row grid, creating shared rhythm
  • Trade-off: Requires knowing the number of rows in advance (grid-row: span 4)

Complexity Comparison

Metric Before After
Lines of CSS ~5 ~12
Shared vertical rhythm No Yes
Fixed row count needed No Yes
Dynamic content support Full Limited

When It Works

  • Fixed structure pages (e.g., landing pages with known sections)
  • Dashboard layouts where panels have consistent structure
  • Magazine layouts with defined column/row grids

When It Doesn't Work

  • Blog posts with variable content length
  • CMS-driven pages where content is dynamic
  • Any case where number of sections varies

Evaluation

Benefit: Creates true vertical rhythm across columns - visually elegant when it works.

Cost: Requires hardcoded row spans. Not suitable for dynamic content.

Recommendation: Marginal benefit for general use. Document as an advanced technique for fixed-structure layouts. Do NOT add to core utilities - the row span requirement limits practical use.

Summary

Subgrid for rows is powerful but inflexible. It's best suited for:

  • Magazine/editorial layouts with designed grid systems
  • Dashboard panels with consistent structure
  • Marketing pages with fixed section counts

For general article layouts with dynamic content, the traditional approach (independent flow) remains more practical.
