Subgrid Demo: Card Alignment

Compare traditional card grids vs subgrid for cross-card alignment of headers, content, and footers.

The Problem

When cards have varying content lengths, their headers and footers don't align across the row. Each card manages its own internal grid independently.

The CSS

Before (Traditional)

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: var(--size-m);
}

.card {
  display: grid;
  grid-template-rows: auto 1fr auto; /* Each card is independent */
}

After (Subgrid)

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  grid-auto-rows: auto 1fr auto; /* Parent defines row tracks */
  gap: var(--size-m);
}

.card {
  display: grid;
  grid-row: span 3;            /* Card spans 3 rows */
  grid-template-rows: subgrid; /* Inherit parent's row tracks */
}

Key Difference

  • Before: Each card has its own grid-template-rows, so row heights are independent
  • After: Parent defines row tracks, cards use subgrid to share them, ensuring alignment

Complexity Comparison

Metric Before After
Lines of CSS ~8 ~10
Cross-card alignment No Yes
HTML changes None None
Browser support All Baseline 2023

Evaluation

Benefit: Subgrid provides true cross-card alignment that's impossible with traditional CSS grid. The alignment is automatic and responds to content changes.

Cost: Minimal - only 2 extra lines of CSS. The conceptual model (cards span rows) is slightly more complex but well worth it.

Recommendation: Add [data-subgrid="rows"] utility to layout-attributes.css for card grids.
