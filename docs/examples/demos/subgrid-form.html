Subgrid Demo: Form Field Alignment

Compare fixed-width labels vs subgrid for cross-fieldset label alignment.

The Problem

Form labels in different fieldsets don't align when using fixed widths. Labels may truncate or waste space depending on the longest label in each fieldset.

Before: Fixed Width Labels

Personal Info
Emergency Contact

↑ Notice: "Emergency Contact Name" and "Emergency Contact Phone" are truncated at 120px!

After: Subgrid Alignment

Personal Info
Emergency Contact

↑ Notice: All labels fully visible! Column auto-sizes to "Emergency Contact Phone".

The CSS

Before (Fixed Width)

.field {
  display: grid;
  grid-template-columns: 120px 1fr; /* Fixed label width */
  gap: var(--size-s);
}

After (Subgrid)

form {
  display: grid;
  grid-template-columns: max-content 1fr; /* Label auto-sizes */
  gap: var(--size-m) var(--size-s);
}

fieldset {
  display: grid;
  grid-column: span 2;
  grid-template-columns: subgrid; /* Inherit form columns */
}

.field {
  display: grid;
  grid-column: span 2;
  grid-template-columns: subgrid; /* Inherit fieldset columns */
}

Key Difference

  • Before: Fixed 120px label column truncates "Emergency Contact Phone" (needs ~170px)
  • After: Form defines max-content 1fr, fieldsets and fields use subgrid to share the column
  • Result: Label column auto-sizes to the longest label across ALL fieldsets - no truncation!

Complexity Comparison

Metric Before After
Lines of CSS ~5 ~15
Cross-fieldset alignment No Yes
Auto-sizing labels No Yes
Nested subgrid levels 0 2

Evaluation

Benefit: Perfect label alignment across fieldsets with auto-sizing. No magic numbers needed.

Cost: More CSS lines, requires understanding nested subgrid (form → fieldset → field).

Recommendation: Beneficial for forms with multiple fieldsets. The extra CSS is worth the alignment and flexibility. Consider adding a [data-layout="form-grid"] utility.
