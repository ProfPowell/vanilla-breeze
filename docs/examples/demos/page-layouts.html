Page Layouts (Opt-in Module)

Semantic HTML page layouts using grid identity. Import separately from the main bundle for automatic layout based on document structure.

Opt-in Import

These layouts require importing two CSS files separately from the main bundle.

@import "vanilla-breeze/layout-grid-identity.css"; @import "vanilla-breeze/layout-page.css";

How It Works

Grid identity automatically assigns grid areas to semantic elements based on their position in the DOM. No classes needed - just use semantic HTML.

/* Automatic grid area assignment */ body > header → grid-area: body-header body > nav → grid-area: body-nav body > main → grid-area: body-main body > aside → grid-area: body-aside body > footer → grid-area: body-footer

page-stack

Mobile-first single column layout. All elements stack vertically.

Header
Link 1 Link 2 Link 3
Main Content
Footer
 <body data-layout="page-stack"> <header>Header</header> <nav>Navigation</nav> <main>Main Content</main> <footer>Footer</footer> </body>

page-sidebar

Two-column layout with navigation sidebar on the left. Responsive: collapses to stack on mobile (unless data-nowrap is set).

Header
Dashboard Projects Analytics Settings
Main Content Area
Footer
 <body data-layout="page-sidebar"> <header>Header</header> <nav>Sidebar Navigation</nav> <main>Main Content</main> <footer>Footer</footer> </body>

page-sidebar-right

Two-column layout with aside on the right. Common for blog posts with related content sidebar.

Header
Main Content Area
Related Sidebar content
Footer
 <body data-layout="page-sidebar-right"> <header>Header</header> <main>Main Content</main> <aside>Sidebar</aside> <footer>Footer</footer> </body>

page-holy-grail

Classic three-column layout with nav, main content, and aside. Responsive: removes aside at 1024px, nav at 768px.

Header
Home About Services Contact
Main Content
Sidebar Related content
Footer
 <body data-layout="page-holy-grail"> <header>Header</header> <nav>Navigation</nav> <main>Main Content</main> <aside>Sidebar</aside> <footer>Footer</footer> </body>

:has() Adaptations

Layouts automatically adapt when elements are missing. No JavaScript required.

page-sidebar without nav

When <nav> is absent, the layout automatically becomes single column.

Header
Main Content (no nav - layout becomes single column)
Footer
 /* CSS :has() automatically adjusts layout */ [data-layout="page-sidebar"]:not(:has(> nav)) { grid-template-columns: 1fr; }

Custom Sidebar Width

Override --sidebar-width to customize the sidebar size.

<body data-layout="page-sidebar" style="--sidebar-width: 320px;"> ... </body>

Comparison: body-* vs page-*

Both systems achieve similar layouts but with different approaches.

Aspect body-* (layout-attributes) page-* (opt-in)
Bundle Included in main.css Separate import
Grid areas Manual assignment available Auto via grid identity
Nested layouts main > aside manual Auto-detected via :has()
Use case Quick prototypes Production apps
