Themes

Vanilla Breeze uses CSS custom properties for a flexible, performant theming system. Beyond colors, themes can transform typography, shapes, shadows, and motion to create distinct design personalities.

Introduction to Theming

The theming system in Vanilla Breeze is built on these core principles:

  1. CSS Custom Properties - All design tokens are defined as CSS variables that can be overridden
  2. OKLCH Color Space - Modern perceptually uniform colors with hue-based theming
  3. Automatic Dark Mode - Uses light-dark() function and color-scheme for native dark mode support
  4. Comprehensive Token Categories - Colors, typography, borders, shadows, and motion can all be themed

Theme Types

Vanilla Breeze includes two types of themes:

  • Color Themes (Ocean, Forest, Sunset) - Override hue values to change the color palette while keeping default shapes and typography
  • Personality Themes (Modern, Minimal, Classic) - Comprehensive themes that transform colors, shapes, shadows, typography, and motion for distinct design feels
/* Color themes override hues */ :root[data-theme="ocean"] { --hue-primary: 200; /* Teal */ --hue-secondary: 180; /* Cyan */ --hue-accent: 45; /* Gold */ } /* Personality themes override multiple token categories */ :root[data-theme="modern"] { /* Colors */ --hue-primary: 270; --color-primary: oklch(55% 0.22 var(--hue-primary)); /* Shape - large radii for soft feel */ --radius-m: 0.75rem; --radius-l: 1rem; /* Shadows - elevated for depth */ --shadow-md: 0 6px 12px -2px oklch(0% 0 0 / 0.12); /* Motion - snappy interactions */ --duration-normal: 150ms; --ease-default: cubic-bezier(0.2, 0, 0.1, 1); }

How Themes Work

The theming system uses a layered approach with CSS custom properties:

1. Base Color Tokens

The base colors.css file defines all color tokens using the default hues. These tokens use OKLCH color space for perceptually uniform colors.

:root { /* Base hues - themes override these */ --hue-primary: 260; --hue-secondary: 200; --hue-accent: 30; /* Primary color with relative color variations */ --color-primary: oklch(50% 0.2 var(--hue-primary)); --color-primary-hover: oklch(from var(--color-primary) calc(l - 0.08) calc(c + 0.02) h); --color-primary-active: oklch(from var(--color-primary) calc(l - 0.12) c h); --color-primary-subtle: oklch(from var(--color-primary) 0.95 0.03 h); /* Interactive colors reference primary */ --color-interactive: var(--color-primary); --color-interactive-hover: var(--color-primary-hover); }

2. Semantic Tokens

Semantic tokens provide meaningful names for common use cases. These use the light-dark() function for automatic dark mode support.

:root { /* Enable light-dark() function */ color-scheme: light dark; /* Surface colors (backgrounds) */ --color-background: light-dark(var(--color-white), var(--color-gray-950)); --color-surface: light-dark(var(--color-white), var(--color-gray-900)); --color-surface-raised: light-dark(var(--color-gray-50), var(--color-gray-800)); --color-surface-sunken: light-dark(var(--color-gray-100), var(--color-gray-950)); /* Text colors */ --color-text: light-dark(var(--color-gray-900), var(--color-gray-100)); --color-text-muted: light-dark(var(--color-gray-600), var(--color-gray-400)); /* Border colors */ --color-border: light-dark(var(--color-gray-200), var(--color-gray-700)); }

3. Theme Overrides

Themes override the hue variables and optionally adjust other tokens. They use the data-theme attribute selector.

/* Theme applied via data-theme attribute */ :root[data-theme="ocean"], [data-theme="ocean"] { --hue-primary: 200; --hue-secondary: 180; --hue-accent: 45; /* Recalculate brand colors with new hues */ --color-primary: oklch(50% 0.15 var(--hue-primary)); --color-primary-hover: oklch(from var(--color-primary) calc(l - 0.08) calc(c + 0.02) h); /* Update interactive to use new primary */ --color-interactive: var(--color-primary); --color-interactive-hover: var(--color-primary-hover); }

4. Dark Mode Combinations

Each theme can have specific dark mode adjustments for optimal contrast and brand cohesion.

/* Theme + Dark mode */ :root[data-theme="ocean"][data-mode="dark"], [data-theme="ocean"][data-mode="dark"] { color-scheme: dark; /* Brighter primary for dark mode contrast */ --color-primary: oklch(60% 0.12 var(--hue-primary)); /* Tinted surfaces for brand cohesion */ --color-background: oklch(12% 0.02 var(--hue-primary)); --color-surface: oklch(16% 0.02 var(--hue-primary)); --color-surface-raised: oklch(20% 0.02 var(--hue-primary)); }

Using the theme-wc Component

The <theme-wc> web component provides a user interface for switching between themes and color modes. It integrates with the ThemeManager for persistence.

Popover Variant (Default)

The popover variant shows a button that opens a theme selection panel when clicked. Great for headers and toolbars.

Try it:

<theme-wc> <button data-trigger class="outline"> <icon-wc name="palette" size="sm"></icon-wc> Change Theme </button> </theme-wc>

Inline Variant

The inline variant displays the theme controls directly without a popover. Ideal for settings pages.

Inline Theme Controls:

<theme-wc data-variant="inline"></theme-wc>

Icon-Only Trigger

For compact interfaces, use an icon-only button with an aria-label.

<theme-wc> <button data-trigger aria-label="Change theme" class="ghost"> <icon-wc name="sun"></icon-wc> </button> </theme-wc>

Component Attributes

Attribute Values Description
data-variant popover (default), inline Display mode for the theme controls
data-open Boolean attribute Whether the popover is open (reflected, popover only)

Dark Mode Support

Vanilla Breeze provides three color modes: Auto, Light, and Dark.

How It Works

  1. Auto Mode - Respects the user's system preference via prefers-color-scheme
  2. Light Mode - Forces light colors regardless of system preference
  3. Dark Mode - Forces dark colors regardless of system preference

The system uses the data-mode attribute on the root element and the color-scheme CSS property:

/* Auto mode (default) - follows system preference */ :root { color-scheme: light dark; } /* Force light mode */ :root[data-mode="light"] { color-scheme: light; } /* Force dark mode */ :root[data-mode="dark"] { color-scheme: dark; /* Adjust primary for better dark mode contrast */ --color-primary: oklch(65% 0.18 var(--hue-primary)); }

The light-dark() Function

Colors that should change between modes use the light-dark() CSS function:

/* Text that adapts to color mode */ --color-text: light-dark(var(--color-gray-900), var(--color-gray-100)); /* Backgrounds */ --color-surface: light-dark(var(--color-white), var(--color-gray-900)); /* Status colors with mode-specific backgrounds */ --color-success-subtle: light-dark( oklch(95% 0.05 145), /* Light mode: pale green */ oklch(25% 0.08 145) /* Dark mode: dark green */ );

Creating a Custom Theme

Follow these steps to create your own brand theme.

Step 1: Choose Your Hues

Select hue values (0-360) for your primary, secondary, and accent colors. Here is a reference for OKLCH hue values:

0-30: Red
30-60: Orange
60-120: Yellow/Green
120-180: Green/Cyan
180-240: Cyan/Blue
240-300: Blue/Purple
300-360: Purple/Red

Step 2: Create the Theme File

Copy the template and customize it for your brand:

/** * Brand Theme: Coral * A warm, friendly coral palette * * Primary: Coral (hue 15) * Secondary: Rose (hue 350) * Accent: Teal (hue 180) */ :root[data-theme="coral"], [data-theme="coral"] { /* Set your hue values (0-360) */ --hue-primary: 15; --hue-secondary: 350; --hue-accent: 180; /* Recalculate brand colors with new hues */ /* Adjust lightness (first number) for visibility */ /* Adjust chroma (second number) for saturation */ --color-primary: oklch(60% 0.18 var(--hue-primary)); --color-primary-hover: oklch(from var(--color-primary) calc(l - 0.08) calc(c + 0.02) h); --color-primary-active: oklch(from var(--color-primary) calc(l - 0.12) c h); --color-primary-subtle: oklch(from var(--color-primary) 0.95 0.03 h); --color-secondary: oklch(55% 0.15 var(--hue-secondary)); --color-secondary-hover: oklch(from var(--color-secondary) calc(l - 0.08) c h); --color-accent: oklch(50% 0.12 var(--hue-accent)); --color-accent-hover: oklch(from var(--color-accent) calc(l - 0.08) c h); /* Update interactive to use new primary */ --color-interactive: var(--color-primary); --color-interactive-hover: var(--color-primary-hover); } /* Theme + Dark mode combination */ :root[data-theme="coral"][data-mode="dark"], [data-theme="coral"][data-mode="dark"] { color-scheme: dark; /* Brighter primary for dark mode contrast */ --color-primary: oklch(65% 0.14 var(--hue-primary)); /* Tinted surfaces for brand cohesion (optional) */ --color-background: oklch(12% 0.02 var(--hue-primary)); --color-surface: oklch(16% 0.02 var(--hue-primary)); --color-surface-raised: oklch(20% 0.02 var(--hue-primary)); }

Step 3: Register the Theme

Add your theme to the themes index file:

@import "./_brand-ocean.css"; @import "./_brand-forest.css"; @import "./_brand-sunset.css"; @import "./_brand-coral.css"; /* Add your new theme */

Step 4: Apply the Theme

Apply your theme using the data-theme attribute:

<!-- On the root element for the whole page --> <html lang="en" data-theme="coral"> <!-- Or on a specific section --> <section data-theme="coral"> <!-- Content with coral theme --> </section>

Applying Themes Programmatically

Use the ThemeManager API for JavaScript-based theme control.

import { ThemeManager } from '/src/lib/theme-manager.js'; // Initialize on page load (reads from localStorage) ThemeManager.init(); // Set color mode ThemeManager.setMode('dark'); // 'auto' | 'light' | 'dark' // Set brand theme ThemeManager.setBrand('ocean'); // Color: 'default' | 'ocean' | 'forest' | 'sunset' // Personality: 'modern' | 'minimal' | 'classic' // Toggle between light and dark ThemeManager.toggleMode(); // Get current state const state = ThemeManager.getState(); // { mode: 'auto', brand: 'ocean', effectiveMode: 'light' } // Reset to defaults ThemeManager.reset(); // Listen for changes window.addEventListener('theme-change', (event) => { const { mode, brand, effectiveMode } = event.detail; console.log(`Theme changed to ${brand} (${effectiveMode} mode)`); });

Direct Attribute Manipulation

You can also set themes directly via DOM attributes:

// Set theme document.documentElement.dataset.theme = 'ocean'; // Set mode document.documentElement.dataset.mode = 'dark'; // Remove (use defaults) delete document.documentElement.dataset.theme; delete document.documentElement.dataset.mode;

Best Practices for Themeable Components

1. Use Semantic Tokens

Always use semantic color tokens rather than raw values. This ensures your components adapt to any theme.

/* Good - adapts to themes */ .card { background: var(--color-surface); border-color: var(--color-border); color: var(--color-text); } /* Avoid - hardcoded colors */ .card { background: #ffffff; border-color: #e5e7eb; color: #1f2937; }

2. Use Interactive Tokens for Clickable Elements

For buttons, links, and other interactive elements, use the interactive color tokens.

.link { color: var(--color-interactive); } .link:hover { color: var(--color-interactive-hover); } .button-primary { background: var(--color-interactive); } .button-primary:hover { background: var(--color-interactive-hover); }

3. Test in Both Modes

Always test your components in both light and dark modes across all themes. Pay attention to:

  • Text contrast ratios (WCAG 2.1 requires 4.5:1 for normal text)
  • Border visibility against backgrounds
  • Shadow intensity (shadows need to be stronger in dark mode)
  • Interactive state visibility

4. Avoid Overriding Theme Tokens

Components should not override theme tokens. Instead, use the existing tokens or create component-specific custom properties that reference theme tokens.

/* Good - component references theme */ .badge { --badge-bg: var(--color-primary-subtle); --badge-text: var(--color-primary); background: var(--badge-bg); color: var(--badge-text); } /* Avoid - overriding theme tokens */ .badge { --color-primary: pink; /* Don't do this! */ }

5. Support Scoped Themes

Remember that themes can be applied to sections, not just the root. Write CSS that works with scoped themes.

<main> <!-- Uses page theme --> <section class="hero">...</section> <!-- Different theme for this section --> <section data-theme="sunset" class="cta"> <!-- All components here use sunset theme --> </section> </main>

Theme File Structure

Themes are organized in the src/tokens/themes/ directory:

src/tokens/ colors.css # Base color tokens with default hues typography.css # Font families, sizes, line heights borders.css # Border widths and radii shadows.css # Shadow scale motion.css # Animation durations and easings themes/ index.css # Imports all theme files _theme-template.css # Template for creating new themes # Color themes _brand-ocean.css # Ocean color theme _brand-forest.css # Forest color theme _brand-sunset.css # Sunset color theme # Personality themes _brand-modern.css # Modern personality theme _brand-minimal.css # Minimal personality theme _brand-classic.css # Classic personality theme

Naming Conventions

  • Theme files are prefixed with _brand-
  • Theme names should be lowercase, hyphenated (e.g., ocean, dark-forest)
  • The data-theme attribute value matches the filename without prefix (e.g., _brand-ocean.css = data-theme="ocean")
