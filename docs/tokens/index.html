Semantic Colors

Context-aware colors that automatically adjust for light and dark modes.

--color-text Primary text
--color-text-muted Secondary text
--color-surface Base background
--color-surface-raised Elevated surface
--color-border Dividers/borders

Interactive Colors

Colors for buttons, links, and interactive elements.

--color-interactive oklch(50% 0.2 260)
--color-interactive-hover oklch(40% 0.2 260)

Usage

.my-element {
  color: var(--color-text);
  background: var(--color-surface);
  border: var(--border-width-thin) solid var(--color-border);
}

.my-button {
  background: var(--color-interactive);
}

.my-button:hover {
  background: var(--color-interactive-hover);
}

Line Heights

--line-height-none (1)
Line height none is useful for single-line text like headings or buttons.
--line-height-tight (1.25)
Tight line height works well for compact UI elements or dense information displays.
--line-height-snug (1.375)
Snug line height provides a balance between tight and normal spacing for body text.
--line-height-normal (1.5)
Normal line height is the default and works well for most body text content.
--line-height-relaxed (1.625)
Relaxed line height improves readability for longer paragraphs of text content.
--line-height-loose (1.75)
Loose line height is best for large text blocks that need maximum readability.

Measure (Line Length)

Token Value Use Case

Spring/Elastic Easings

Token Description
--ease-elastic-1 Subtle bounce at end
--ease-elastic-2 More pronounced bounce
--ease-squish-1 Anticipation + overshoot
--ease-squish-2 Exaggerated squish effect

Reduced Motion

All duration tokens automatically become 0ms when prefers-reduced-motion: reduce is set, ensuring accessibility for users who are sensitive to motion.

Usage

.button {
  transition:
    background-color var(--duration-fast) var(--ease-default),
    transform var(--duration-normal) var(--ease-out);
}

.button:hover {
  transform: translateY(-2px);
}

.modal {
  animation: fade-in var(--duration-normal) var(--ease-out);
}
