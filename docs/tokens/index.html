Colors

Color tokens use OKLCH color space for perceptually uniform colors with automatic dark mode support.

Gray Scale

Neutral grays from light to dark, used for text, backgrounds, and borders.

--color-gray-50 oklch(98% 0 0) --color-gray-100 oklch(96% 0 0) --color-gray-200 oklch(90% 0 0) --color-gray-300 oklch(80% 0 0) --color-gray-400 oklch(70% 0 0) --color-gray-500 oklch(55% 0 0) --color-gray-600 oklch(45% 0 0) --color-gray-700 oklch(35% 0 0) --color-gray-800 oklch(25% 0 0) --color-gray-900 oklch(15% 0 0)

Semantic Colors

Context-aware colors that automatically adjust for light and dark modes.

--color-text Primary text --color-text-muted Secondary text --color-surface Base background --color-surface-raised Elevated surface --color-border Dividers/borders

Interactive Colors

Colors for buttons, links, and interactive elements.

--color-interactive oklch(50% 0.2 260) --color-interactive-hover oklch(40% 0.2 260)

Usage

.my-element { color: var(--color-text); background: var(--color-surface); border: var(--border-width-thin) solid var(--color-border); } .my-button { background: var(--color-interactive); } .my-button:hover { background: var(--color-interactive-hover); }

Spacing

A consistent spacing scale using t-shirt sizes, built on a 0.25rem (4px) base unit.

--size-3xs 0.125rem (2px) --size-2xs 0.25rem (4px) --size-xs 0.5rem (8px) --size-s 0.75rem (12px) --size-m 1rem (16px) --size-l 1.5rem (24px) --size-xl 2rem (32px) --size-2xl 3rem (48px) --size-3xl 4rem (64px)

Base Unit

Token Value Description
--size-unit 0.25rem Base unit for spacing calculations

Usage

.card { padding: var(--size-l); margin-block-end: var(--size-m); gap: var(--size-s); }

Line Heights

--line-height-none (1) Line height none is useful for single-line text like headings or buttons. --line-height-tight (1.25) Tight line height works well for compact UI elements or dense information displays. --line-height-snug (1.375) Snug line height provides a balance between tight and normal spacing for body text. --line-height-normal (1.5) Normal line height is the default and works well for most body text content. --line-height-relaxed (1.625) Relaxed line height improves readability for longer paragraphs of text content. --line-height-loose (1.75) Loose line height is best for large text blocks that need maximum readability.

Measure (Line Length)

Token Value Use Case

Letter Spacing

Token Value Use Case
--letter-spacing-tight -0.025em Large headings

--letter-spacing-wide 0.025em Small text, labels
--letter-spacing-wider 0.05em Uppercase text

Usage

.heading { font-family: var(--font-sans); font-size: var(--font-size-3xl); font-weight: var(--font-weight-bold); line-height: var(--line-height-tight); letter-spacing: var(--letter-spacing-tight); } .body-text { font-size: var(--font-size-md); line-height: var(--line-height-normal); max-inline-size: var(--measure-normal); }

Motion

Duration and easing tokens for smooth animations. Automatically disabled for users who prefer reduced motion.

Durations

--duration-instant 50ms --duration-fast 100ms --duration-normal 200ms --duration-slow 300ms --duration-slower 500ms

Easing Functions

Standard easing curves for natural-feeling motion.

--ease-1 (Subtle) --ease-2 (Moderate) --ease-3 (Standard) --ease-4 (Emphasized) --ease-5 (Dramatic)

Spring/Elastic Easings

Token Description
--ease-elastic-1 Subtle bounce at end
--ease-elastic-2 More pronounced bounce
--ease-squish-1 Anticipation + overshoot
--ease-squish-2 Exaggerated squish effect

Reduced Motion

All duration tokens automatically become 0ms when prefers-reduced-motion: reduce is set, ensuring accessibility for users who are sensitive to motion.

Usage

.button { transition: background-color var(--duration-fast) var(--ease-default), transform var(--duration-normal) var(--ease-out); } .button:hover { transform: translateY(-2px); } .modal { animation: fade-in var(--duration-normal) var(--ease-out); }
