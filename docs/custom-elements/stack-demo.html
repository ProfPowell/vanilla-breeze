layout-stack (Enhanced Documentation Demo)

This page demonstrates enhanced documentation using <browser-window> and <code-block> for a CSS-only custom element.

Attributes

Attribute Values Default Description
data-gap none, 3xs...3xl m Vertical gap between children
data-align start, center, end, stretch stretch Horizontal alignment

Basic Usage

The stack creates vertical rhythm with consistent spacing. Click the code icon to view source, or double-click the title bar to maximize.

<layout-stack data-gap="m"> <div>First item</div> <div>Second item</div> <div>Third item</div> </layout-stack>

Gap Variants

Compare different gap sizes. The data-gap attribute accepts t-shirt sizes:

<layout-stack data-gap="xs"> <!-- Extra small gap (0.5rem) --> </layout-stack> <layout-stack data-gap="m"> <!-- Medium gap (1rem) - default --> </layout-stack> <layout-stack data-gap="xl"> <!-- Extra large gap (2rem) --> </layout-stack>

Gap Scale Reference

--size-3xs: 0.125rem; /* 2px */ --size-2xs: 0.25rem; /* 4px */ --size-xs: 0.5rem; /* 8px */ --size-s: 0.75rem; /* 12px */ --size-m: 1rem; /* 16px - default */ --size-l: 1.5rem; /* 24px */ --size-xl: 2rem; /* 32px */ --size-2xl: 3rem; /* 48px */ --size-3xl: 4rem; /* 64px */

Alignment Options

Control horizontal alignment of stack children:

<layout-stack data-gap="s" data-align="start"> <button>Aligned to start</button> </layout-stack> <layout-stack data-gap="s" data-align="center"> <button>Centered</button> </layout-stack> <layout-stack data-gap="s" data-align="end"> <button>Aligned to end</button> </layout-stack>

Real-World Example: Card Content

Stack is commonly used inside cards to organize content. View the source or open in CodePen:

<layout-card data-variant="elevated" data-padding="m"> <layout-stack data-gap="s"> <h3>Card Title</h3> <p>A description of this card's content.</p> <layout-cluster data-gap="s"> <button>Action</button> <button class="secondary">Cancel</button> </layout-cluster> </layout-stack> </layout-card>

CSS Implementation

The stack is implemented with pure CSS using flexbox - no JavaScript required:

layout-stack { display: flex; flex-direction: column; gap: var(--size-m); } layout-stack[data-gap="none"] { gap: 0; } layout-stack[data-gap="xs"] { gap: var(--size-xs); } layout-stack[data-gap="s"] { gap: var(--size-s); } layout-stack[data-gap="m"] { gap: var(--size-m); } layout-stack[data-gap="l"] { gap: var(--size-l); } layout-stack[data-gap="xl"] { gap: var(--size-xl); } layout-stack[data-align="start"] { align-items: flex-start; } layout-stack[data-align="center"] { align-items: center; } layout-stack[data-align="end"] { align-items: flex-end; } layout-stack[data-align="stretch"] { align-items: stretch; }

Nested Stacks

Stacks can be nested with different gap sizes:

<layout-stack data-gap="l"> <h2>Section Title</h2> <layout-stack data-gap="s"> <p>First paragraph with tighter spacing.</p> <p>Second paragraph.</p> </layout-stack> <layout-stack data-gap="xs"> <small>Fine print line 1</small> <small>Fine print line 2</small> </layout-stack> </layout-stack>

Key Differences: CSS-Only vs Web Component

Unlike web components like tabs-wc, layout-stack requires no JavaScript:

Feature layout-stack (CSS-only) tabs-wc (Web Component)
JavaScript required No For ARIA/keyboard
Configuration data-attributes data-attributes + ARIA
Styling CSS selectors CSS + Shadow DOM aware
Events None needed Custom events (tab-change)
