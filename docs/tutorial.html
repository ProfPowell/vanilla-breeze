Learn Vanilla Breeze step by step. In this tutorial, you will build a complete landing page from scratch, mastering layout primitives, design tokens, forms, and interactive components along the way.

1. Introduction and Philosophy

Before we write any code, let us understand what makes Vanilla Breeze different from other CSS frameworks.

What We Will Build

By the end of this tutorial, you will have built a complete landing page with:

  • A hero section with a call-to-action
  • A responsive features grid
  • Styled cards with proper elevation
  • Navigation with multiple styles
  • A contact form with validation
  • An interactive FAQ accordion

The Vanilla Breeze Philosophy

HTML-First

Start with semantic HTML5 elements. Your page works before any CSS loads.

Progressive Enhancement

Add CSS for styling, JavaScript for interactivity. Each layer is optional.

Zero Dependencies

No build tools required. Link a CSS file and you are ready to go.

Composition Over Configuration

Combine simple layout primitives to build complex interfaces.

Key Concept: Vanilla Breeze uses three layers of components: Native Elements (84 styled HTML5 elements), Custom Elements (15 CSS-only layout primitives), and Web Components (9 interactive JavaScript components). You only use what you need.

2. Setting Up the Project

Let us create our project structure and link the Vanilla Breeze stylesheet.

File Structure

Create a new folder for your project with this structure:

my-landing-page/ index.html

That is it. No node_modules, no build configuration, no package.json required.

The Minimal HTML Document

Create your index.html file with this starting template:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My Landing Page</title> <!-- Vanilla Breeze CSS - that's all you need! --> <link rel="stylesheet" href="https://unpkg.com/vanilla-breeze/dist/vanilla-breeze.css"> </head> <body> <h1>Hello, Vanilla Breeze!</h1> <p>Your page is ready.</p> </body> </html>

Open this file in your browser and you will see styled typography immediately. The CSS handles heading styles, paragraph spacing, and sensible defaults for all HTML elements.

Pro Tip: For local development, you can download the CSS file or use a CDN. For production, consider self-hosting for better performance and reliability.

4. Design Tokens and Theming

Vanilla Breeze uses CSS custom properties (design tokens) for consistent spacing, colors, and typography. Understanding these is key to customization.

The Token System

All design values come from tokens. Here are the most commonly used ones:

Spacing (T-shirt Sizes)

/* Spacing scale */ --size-3xs: 0.125rem; /* 2px */ --size-2xs: 0.25rem; /* 4px */ --size-xs: 0.5rem; /* 8px */ --size-s: 0.75rem; /* 12px */ --size-m: 1rem; /* 16px */ --size-l: 1.5rem; /* 24px */ --size-xl: 2rem; /* 32px */ --size-2xl: 3rem; /* 48px */ --size-3xl: 4rem; /* 64px */

Colors (Semantic)

/* Surface colors */ --color-surface: white; --color-surface-raised: #f9f9f9; --color-background: white; /* Text colors */ --color-text: #171717; --color-text-muted: #6b6b6b; /* Interactive */ --color-interactive: #6366f1; --color-interactive-hover: #4f46e5; /* Borders */ --color-border: #e5e5e5;

Customizing Your Theme

To customize colors, override the hue variables. Add this to your <head>:

<style> :root { /* Change the primary hue (0-360 on color wheel) */ --hue-primary: 200; /* Cyan/Blue */ /* Or set specific colors */ --color-interactive: oklch(50% 0.2 200); } </style>

Using Tokens in Your CSS

When writing custom styles, always use tokens instead of hard-coded values:

.my-custom-section { /* Good - uses tokens */ padding: var(--size-xl); background: var(--color-surface-raised); border-radius: var(--radius-m); /* Avoid - hard-coded values */ /* padding: 32px; */ /* background: #f5f5f5; */ }
Why Tokens? Using tokens ensures your customizations automatically work with dark mode, maintain consistency, and make global changes easy. Change one token value, and it updates everywhere.

5. Building the Hero Section

Now let us create an impactful hero section using layout-stack and layout-cluster - two powerful layout primitives.

Understanding layout-stack

The <layout-stack> element creates a vertical stack with consistent spacing. It is the most commonly used layout primitive.

<layout-stack data-gap="m"> <h1>Stacked vertically</h1> <p>With consistent gaps</p> <p>Between each element</p> </layout-stack>

Understanding layout-cluster

The <layout-cluster> element arranges items horizontally with wrapping. Perfect for button groups and inline elements.

<layout-cluster data-gap="s"> <button>Primary</button> <button class="secondary">Secondary</button> </layout-cluster>

Building the Hero

Combine these primitives to create a compelling hero section:

<main> <section class="hero"> <layout-stack data-gap="l" data-align="center"> <h1>Build Better Products</h1> <p class="lead"> The all-in-one platform that helps modern teams ship faster with confidence. </p> <layout-cluster data-gap="s" data-justify="center"> <button>Get Started Free</button> <button class="secondary">Watch Demo</button> </layout-cluster> </layout-stack> </section> </main>

Add some custom styling for the hero:

<style> .hero { padding-block: var(--size-3xl); text-align: center; } .hero h1 { font-size: clamp(var(--font-size-2xl), 5vw, var(--font-size-4xl)); margin: 0; } .hero .lead { font-size: var(--font-size-lg); color: var(--color-text-muted); max-inline-size: 50ch; margin: 0; } </style>

Data Attributes Reference

Both layout elements accept these data attributes:

Attribute Values Description
data-gap xs, s, m, l, xl Space between items
data-align start, center, end, stretch Cross-axis alignment
data-justify start, center, end, between Main-axis alignment (cluster only)

6. Creating a Features Grid

Display your product features in a responsive grid that adapts to any screen size.

Understanding layout-grid

The <layout-grid> element creates a responsive grid that automatically adjusts columns based on available space. No media queries needed.

<layout-grid data-min="15rem" data-gap="l"> <!-- Items automatically flow into columns --> <div>Item 1</div> <div>Item 2</div> <div>Item 3</div> </layout-grid>

The data-min attribute sets the minimum width of each column. The grid calculates how many columns fit automatically.

Building the Features Section

Add this below your hero section:

<section id="features" class="features"> <layout-stack data-gap="xl"> <layout-stack data-gap="s" data-align="center"> <h2>Everything You Need</h2> <p class="section-lead">Powerful features to help your team succeed.</p> </layout-stack> <layout-grid data-min="15rem" data-gap="l"> <div class="feature"> <h3>Lightning Fast</h3> <p>Built for speed. Your pages load instantly.</p> </div> <div class="feature"> <h3>Secure by Default</h3> <p>Enterprise-grade security out of the box.</p> </div> <div class="feature"> <h3>Team Collaboration</h3> <p>Work together seamlessly with your team.</p> </div> <div class="feature"> <h3>Analytics Built-in</h3> <p>Understand your users with powerful insights.</p> </div> <div class="feature"> <h3>API Access</h3> <p>Integrate with any tool your team uses.</p> </div> <div class="feature"> <h3>24/7 Support</h3> <p>Our team is here to help anytime.</p> </div> </layout-grid> </layout-stack> </section>

Add styles for the features section:

.features { padding-block: var(--size-3xl); background: var(--color-surface-raised); } .section-lead { color: var(--color-text-muted); max-inline-size: 50ch; text-align: center; } .feature { padding: var(--size-l); background: var(--color-surface); border-radius: var(--radius-m); box-shadow: var(--shadow-sm); } .feature h3 { font-size: var(--font-size-lg); margin-block-end: var(--size-xs); } .feature p { color: var(--color-text-muted); margin: 0; }

7. Building Card Components

Cards are a fundamental UI pattern. Vanilla Breeze provides a layout-card element with built-in variants.

Understanding layout-card

The <layout-card> element provides a container with padding, background, and optional elevation styles.

<!-- Default card with subtle shadow --> <layout-card> <h3>Default Card</h3> <p>Subtle shadow, rounded corners.</p> </layout-card> <!-- Elevated card --> <layout-card data-variant="elevated"> <h3>Elevated Card</h3> <p>More prominent shadow.</p> </layout-card> <!-- Outlined card --> <layout-card data-variant="outlined"> <h3>Outlined Card</h3> <p>Border instead of shadow.</p> </layout-card>

Pricing Cards Example

Let us add a pricing section using cards:

<section id="pricing" class="pricing"> <layout-stack data-gap="xl"> <layout-stack data-gap="s" data-align="center"> <h2>Simple Pricing</h2> <p class="section-lead">Choose the plan that's right for you.</p> </layout-stack> <layout-grid data-min="18rem" data-gap="l"> <layout-card data-variant="outlined"> <layout-stack data-gap="m"> <h3>Starter</h3> <p class="price"><strong>$9</strong>/month</p> <ul> <li>5 projects</li> <li>Basic analytics</li> <li>Email support</li> </ul> <button class="secondary">Get Started</button> </layout-stack> </layout-card> <layout-card data-variant="elevated"> <layout-stack data-gap="m"> <h3>Pro</h3> <p class="price"><strong>$29</strong>/month</p> <ul> <li>Unlimited projects</li> <li>Advanced analytics</li> <li>Priority support</li> </ul> <button>Get Started</button> </layout-stack> </layout-card> <layout-card data-variant="outlined"> <layout-stack data-gap="m"> <h3>Enterprise</h3> <p class="price"><strong>Custom</strong></p> <ul> <li>Everything in Pro</li> <li>Custom integrations</li> <li>Dedicated support</li> </ul> <button class="secondary">Contact Sales</button> </layout-stack> </layout-card> </layout-grid> </layout-stack> </section>

8. Adding Navigation

Vanilla Breeze provides multiple navigation styles through simple class names.

Navigation Variants

The <nav> element supports several style variants:

<!-- Horizontal nav with pills (default for headers) --> <nav class="horizontal pills"> <ul> <li><a href="#" aria-current="page">Home</a></li> <li><a href="#">Features</a></li> <li><a href="#">Pricing</a></li> </ul> </nav> <!-- Tab-style navigation --> <nav class="horizontal tabs"> <ul> <li><a href="#" aria-current="page">Overview</a></li> <li><a href="#">Settings</a></li> <li><a href="#">Billing</a></li> </ul> </nav> <!-- Minimal nav for footers --> <nav class="horizontal minimal"> <ul> <li><a href="#">Privacy</a></li> <li><a href="#">Terms</a></li> <li><a href="#">Contact</a></li> </ul> </nav>

Updated Header

Let us enhance our header with proper navigation styling:

9. Forms and Inputs

Vanilla Breeze styles all native form elements automatically. Let us build a contact form.

Form Layout Patterns

Forms support several layout classes:

/* Stack form fields vertically (default) */ form.stacked { ... } /* Inline form for search bars */ form.inline { ... } /* Two-column grid layout */ form.grid { ... }

Building the Contact Form

Add a contact section with a stacked form:

Form Validation States

Native HTML5 validation is styled automatically:

<!-- Required field - shows error when empty on submit --> <input type="email" required> <!-- Pattern validation --> <input type="text" pattern="[A-Za-z]+" title="Letters only"> <!-- Help text and error messages --> <div class="group"> <label for="password">Password</label> <input type="password" id="password" minlength="8" required> <span class="help">Must be at least 8 characters</span> </div>

10. Adding Interactivity

For interactive components, Vanilla Breeze provides Web Components. Let us add an FAQ accordion.

Including JavaScript

Add the JavaScript file before your closing </body> tag:

<!-- Just before </body> --> <script type="module" src="https://unpkg.com/vanilla-breeze/dist/vanilla-breeze.js"></script>

The Accordion Component

The <accordion-wc> component enhances native <details> elements with keyboard navigation and optional single-open mode.

Sign up for a free account, and you'll be guided through our onboarding process. No credit card required.</p> </div> </details> <details name="faq"> <summary>Can I cancel anytime?</summary> <div> <p>Yes, you can cancel your subscription at any time. There are no long-term contracts or cancellation fees.</p> </div> </details> <details name="faq"> <summary>Do you offer a free trial?</summary> <div> <p>We offer a 14-day free trial on all paid plans. 11. Responsive and Accessible

Vanilla Breeze is designed to be responsive and accessible by default. Let us review best practices.

Built-in Responsiveness

The layout primitives are inherently responsive:

  • layout-grid: Automatically adjusts columns based on data-min
  • layout-cluster: Wraps items naturally when space runs out
  • layout-stack: Works at any width
  • layout-sidebar: Stacks on narrow viewports

Fluid Typography

Use CSS clamp() for headings that scale smoothly:

.hero h1 { /* Minimum 1.5rem, preferred 5vw, maximum 3rem */ font-size: clamp(var(--font-size-2xl), 5vw, var(--font-size-4xl)); }

Touch Targets

All interactive elements have a minimum size of 44x44 pixels, meeting WCAG 2.1 touch target guidelines:

/* Built into Vanilla Breeze */ --size-touch-min: 2.75rem; /* 44px */ button, input, select { min-block-size: var(--size-touch-min); }

Color Contrast

The OKLCH color system ensures proper contrast ratios. Text and interactive elements meet WCAG AA requirements.

Accessibility Checklist

When building with Vanilla Breeze, ensure you:

  • Use semantic HTML elements (<nav>, <main>, <section>)
  • Add aria-label to navigation landmarks
  • Use aria-current="page" for current navigation items
  • Include alt text for images
  • Ensure form fields have associated <label> elements
  • Test with keyboard navigation (Tab, Enter, Arrow keys)

Dark Mode Support

Dark mode works automatically through CSS light-dark() functions:

/* Colors adapt automatically to user preference */ --color-surface: light-dark(white, #171717); --color-text: light-dark(#171717, #f5f5f5); /* Force a specific mode if needed */ :root[data-mode="dark"] { color-scheme: dark; }
Test Early: Test your page with keyboard navigation and a screen reader (VoiceOver on Mac, NVDA on Windows) during development, not after.
