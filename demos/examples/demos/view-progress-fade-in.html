Scroll down — each card fades in as it enters the viewport

The opacity is driven by view progress, not a one-shot event. Scroll halfway down a card and the card is halfway opaque.

First reveal

This card uses data-effect="fade-in" data-trigger="view-progress". As it enters the viewport, opacity climbs from 0 to 1 in lockstep with scroll. Stop scrolling and the opacity stays where it stopped.

Continuous, not binary

Compare with the old IntersectionObserver-based data-trigger="scroll" — that fires once at 10% visible and plays a fixed-duration animation regardless of scroll speed. This one tracks scroll.

Zero JS

This entire effect is pure CSS via animation-timeline: view(). Browsers without support (Firefox without the flag) see the card in its idle hidden state — add data-trigger="scroll view-progress" for a one-shot fallback.

Composes with the slot system

Every entrance effect (fade-in, slide-up, slide-in, pop) works under view-progress. The slot composer handles cross-slot pairs as usual.
