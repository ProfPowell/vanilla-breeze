Mermaid Diagrams

Mermaid blocks are rendered as fenced code by marked. A post-render listener lazy-loads Mermaid only when a diagram is present.

# Project Architecture

A flowchart showing the build pipeline:

```mermaid
graph TD
    A[Source Files] --> B[esbuild]
    B --> C[CDN Bundle]
    B --> D[Component JS]
    C --> E[Production Site]
    D --> E
    E --> F[Browser]
```

## Sequence Diagram

How the markdown-viewer renders content:

```mermaid
sequenceDiagram
    participant Browser
    participant Component
    participant Marked

    Browser->>Component: connectedCallback()
    Component->>Component: resolveMarkdown()
    Component->>Marked: parse(md)
    Marked-->>Component: HTML string
    Component->>Browser: innerHTML = html
    Component->>Browser: dispatch rendered event
```

## Regular Code Block

Standard code blocks are unaffected:

```javascript
// This stays as a code block, not a diagram
console.log('Hello from markdown-viewer');
```
