The Evolution of Digital Reading

February 28, 2026

The way we read has changed dramatically over the past two decades. From physical books to e-readers to web articles, each medium has brought its own affordances and limitations. Yet one thing remains constant: readers want to engage with text, not just consume it passively.

Highlighting and annotating have been fundamental reading activities since the invention of writing. Medieval scholars filled manuscript margins with glosses. Students have long wielded highlighter pens. And in the digital age, tools like Medium, Kindle, and Hypothesis have brought these practices online.

The CSS Custom Highlight API represents a significant advancement for web-based annotation. Unlike earlier approaches that required wrapping text in <span> or <mark> elements — which could break the DOM structure and interfere with other page functionality — the Highlight API paints directly over existing text nodes without modifying the document.

This non-destructive approach means highlights can coexist peacefully with other text enhancements: links, emphasis, code snippets, and interactive widgets all remain fully functional. The browser composites the highlight layer on top during painting, much like a physical highlighter pen works on printed paper.

Local storage provides a natural home for personal annotations. Unlike server-side solutions that raise privacy concerns and require authentication, localStorage keeps notes private to the user and device. The trade-off — no cross-device sync — is acceptable for many use cases, especially documentation sites and study materials.
