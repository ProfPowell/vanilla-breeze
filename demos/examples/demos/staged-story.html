Staged story pattern

One pinned visual on the left. Sequential narrative sections on the right. Pure CSS — no scroll listeners, no IntersectionObserver. Resize narrow to see the layout collapse to a single column with no sticky pinning.

Introduce the subject

The first step sets the scene. The pinned figure stays in view while readers move through the opening paragraph at their own pace.

Copy is the load-bearing element here. The visual is supporting context, not the message.

Reveal the tension

A second step advances the narrative. The pinned figure remains visible, so the visual context is continuous even though the words progress.

This is the pattern's reason for existing: keeping context anchored while the reader moves through a sequence.

Resolve and exit

The closing step lets the reader land. After this step, the figure releases from its pin and the layout returns to normal flow.

Anchor links inside steps use scroll-margin-block-start so deep links land below any fixed UI rather than under it.

What this pattern is not

This is a layout pattern, not an animation system. The copy reveal inside each step is handled by the existing data-animate attribute — no new motion code was needed.

If animation-timeline: view() is unsupported, the reveal animations disable and the pinned layout still works as plain sticky CSS. The page remains readable end-to-end.

Sticky pinning works everywhere. Step reveals require animation-timeline: view() support (Chrome 115+, Safari 18+).
