data-accept — Character Filter

Restrict input to allowed characters without enforcing a rigid format. Users type freely — only invalid characters are blocked.

Presets

Phone

Allows: digits, +, (, ), -, space

Date

Allows: digits, /, -, .

Credit Card

Allows: digits, spaces, -

Currency

Allows: digits, commas, periods

Digits Only

Allows: digits only

Alpha Only

Allows: letters only

Alphanumeric

Allows: letters and digits

Hex

Allows: 0-9, a-f, A-F

Custom Character Class

Custom regex: [0-9a-fA-F]

Same as the hex preset, using custom bracket syntax.

Paste Behavior

Try pasting mixed content

Paste abc123def456 — result: 123456. Invalid characters are stripped, valid ones kept.

data-accept vs data-mask

Both filter input, but they serve different purposes:

data-accept="phone" (flexible)

User decides the format. Only invalid chars blocked.

data-mask="phone" (rigid)

Format enforced: (###) ###-####. Separators auto-injected.

Dynamic Elements

Inputs added via JS are auto-enhanced by the shared MutationObserver.
