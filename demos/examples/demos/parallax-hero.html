Built on motion that respects readers.

The background drifts at a different rate than this copy as you scroll. The text never leaves a static, readable layer.

What this demo proves

The hero uses a single data-parallax media layer driven by a section-local view() timeline. There is no JavaScript. There are no scroll listeners.

On browsers without scroll-driven animation support, the hero figure stays static and the layout remains correct. The foreground content is always readable; nothing is locked to motion to make sense.

How to tune it

Three custom properties drive the effect — set them inline or in your own CSS:

  • --vb-parallax-from (default 0%) — start translateY
  • --vb-parallax-to (default -12%) — end translateY
  • --vb-parallax-scale (default 1) — apply a small scale so the layer can drift without revealing edges

Keep the range subtle. Heavy parallax breaks rhythm and quickly degrades on smaller viewports.

Reduced motion

When the user has prefers-reduced-motion: reduce set, the animation is removed and the figure renders at its starting transform. The static hero remains intentional — the copy still does its job.

Where this fits

Use data-parallax for at most one media layer per section. For staged storytelling with pinned visuals and sequential narrative steps, see data-stage instead.

Parallax requires animation-timeline: view() support (Chrome 115+, Safari 18+). Other browsers see the static hero.
