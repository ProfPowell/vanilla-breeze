version-switcher — Phase 3

Three Phase-3 features: meta-tag manifest fallback, archived-version banner, and mounting inside a <page-info> panel.

1. Meta-tag manifest fallback

This page has no inline data-versions on the switcher, but the <head> contains <meta name="vb:versions-manifest" content="…json">. The component fetches that URL and resolves the version list from it.

2. Archived-version banner

The page declares <meta itemprop="version" content="v1.0">; v1.0 is archived in this manifest (which has no explicit current: true, so the meta tag wins). With data-banner set, the switcher renders a warning banner pointing at the latest non-archived entry.

3. Mount inside <page-info>

The switcher can render its version list as a section inside a <page-info>'s expandable panel via data-page-info-target="ID". The standalone trigger is hidden in this mode.
