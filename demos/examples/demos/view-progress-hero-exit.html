Scroll to see me leave

This hero uses animation-timeline: view() with animation-range: exit to scale down and fade as it leaves the top of the viewport.

Below the fold

The hero above is a custom animation-timeline use case. The same machinery powers VB's data-trigger="view-progress:exit" for any entrance effect — wire data-effect="fade-out" data-trigger="view-progress:exit" on any element for an exit-driven fade.

Scroll back up to reverse the effect. Stop halfway and the hero stops halfway. That's the difference between scroll-driven animation and a one-shot trigger.

The fallback for browsers without animation-timeline support is the static end state — the hero just stays full-size. No broken layout, no flashing. Progressive enhancement at the platform level.
