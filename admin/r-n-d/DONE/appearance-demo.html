Vanilla Breeze

CSS appearance — Evaluation Demo

What this demos: The appearance CSS property strips browser-native widget chrome, giving VB full control over form element styling via design tokens. Each section shows Native → Bare (appearance:none) → VB Styled. The bare column reveals what appearance:none removes; the VB column shows how tokens then govern the result.

Checkbox

<input type="checkbox">
Browser Native
appearance: none (bare)
appearance: none;
/* All native chrome removed — completely invisible without styling */
VB Styled
--control-checked-bg: var(--color-primary);
--control-radius-check: var(--radius-s);
/* :checked::after uses clip-path checkmark */
✓ Include Recommendation
Core VB pattern. appearance:none is the only cross-browser way to fully control checkbox geometry. The clip-path checkmark technique avoids SVG dependencies entirely — pure CSS, zero assets. Both :checked and :indeterminate states are reachable. --control-size, --control-checked-bg, --control-radius-check are the three tokens themes need to override.

Radio Button

<input type="radio">
Browser Native
appearance: none (bare)
/* Invisible — border-radius:50% already enforced by spec */
VB Styled
--control-radius-radio: var(--radius-full);
/* :checked::after = inner dot via inset */
✓ Include Recommendation
Same pattern as checkbox. The only difference is border-radius: var(--radius-full) and the ::after dot uses inset: 3px instead of clip-path. Token separation of checkbox and radio via --control-radius-check vs --control-radius-radio lets themes make squares, pills, or soft-squares independently.

Select / Dropdown

<select>
Browser Native
appearance: none (bare)
/* Chevron gone — border/bg must be provided */
VB Styled
/* Chevron = inline SVG data-URI in background shorthand */
--select-chevron: url(…svg…);
/* Opens native OS picker — no JS needed */
✓ Include Recommendation
The highest-value appearance:none use in form styling. Removes the OS chrome arrow and bevels, leaving a plain box. VB injects an inline SVG chevron via background shorthand — no extra elements, no pseudo-elements, and the dropdown itself remains a native <select> (accessible, keyboard-navigable, opens system picker on mobile). Token: --select-chevron is a data-URI so themes can recolor it.

Text, Number & Search Inputs

<input type="text|number|search">
Browser Native
What gets removed
/* Number — removes spinner arrows: */
::-webkit-inner-spin-button,
::-webkit-outer-spin-button {
  appearance: none;
}
-moz-appearance: textfield;

/* Search — removes OS × cancel icon: */
::-webkit-search-cancel-button {
  appearance: none;
}

/* Textarea — removes resize handle: */
resize: vertical;
VB Styled
/* Number: no spinners, styled as textfield */
/* Search: no system cancel button chrome */
--input-height: 2.5rem;
--input-radius: var(--radius-m);
✓ Include Recommendation
Pseudo-element appearance resets, not the element itself. For number and search, VB targets the internal pseudo-elements (::-webkit-inner-spin-button, ::-webkit-search-cancel-button) rather than the input. This is the correct progressive enhancement approach — removing OS affordances that conflict with VB's own icon/action system. Firefox needs -moz-appearance: textfield as the parallel reset.

Range Slider

<input type="range">
Browser Native
appearance: none (bare)
/* Track and thumb both removed — nothing renders */
VB Styled
--range-track-h: 0.375rem;
--range-thumb-size: 1.25rem;
/* Fill via CSS gradient (no JS for track color) */
/* Variant: override --range-track-fill inline */
△ Caveat Recommendation
Include, but document the gradient limitation. The track fill uses a CSS gradient trick (background: linear-gradient(to right, fill 0% var(--_pct), track var(--_pct) 100%)). For the fill to track the thumb position, --_pct must be updated on input. VB Layer 4 JS handles this. Layer 2-3 (CSS-only) gets the styled track/thumb but a static fill percentage. This is an honest progressive enhancement — functional at all layers, enhanced by JS.

Progress Bar

<progress>
Browser Native
value=65 · indeterminate
appearance: none (bare)
/* No track or fill — must style ::-webkit-progress-bar */
/* and ::-webkit-progress-value separately */
VB Styled
data-variant: "success|warning|error"
--progress-fill: var(--color-primary);
/* Variants override --progress-fill per token */
✓ Include Recommendation
Semantic element, full token control. <progress> has a value attribute that assistive technology reads as a percentage — semantically correct vs a ARIA roleplay div. appearance:none + pseudo-element selectors (::-webkit-progress-bar, ::-moz-progress-bar) give full cross-browser control. The data-variant attribute fits VB's data-attribute state model perfectly.

Meter

<meter>
Browser Native
optimum · suboptimum · poor
appearance: none (bare)
/* Semantic low/high/optimum ranges — */
/* browser auto-colours green/yellow/red */
/* appearance:none removes all colour logic */
VB Styled
::-webkit-meter-optimum-value
  → var(--color-success)
::-webkit-meter-suboptimum-value
  → var(--color-warning)
::-webkit-meter-even-less-good-value
  → var(--color-error)
△ Caveat Recommendation
Include for completeness; Firefox parity is partial. Chromium-based browsers expose the three pseudo-elements for optimum/sub/poor coloring. Firefox only exposes ::-moz-meter-bar (no semantic colour splitting). The semantic value (low, high, optimum attributes) still reads correctly via AT regardless of visual treatment. VB can map to status tokens and accept the Firefox limitation.

File Input

<input type="file">
Browser Native
appearance: none limitation
/* File input resists appearance:none */
/* The button + text area are UA-internal */
/* Correct approach: visually-hidden input */
/* + styled <label> as the visible target */

::file-selector-button {
  appearance: none;
  /* Styles just the button portion */
}
VB Pattern (label wrap)
/* Input: visually hidden (opacity:0, 0px) */
/* Label: styled as a button, proxies click */
/* Input keeps focus — :focus-visible on input */
/* triggers outline on label via + selector */
△ Workaround Recommendation
appearance:none on file inputs is unreliable — use the label-wrap pattern. The visually-hidden input + visible label is the correct VB pattern: native file picker opens on label click, keyboard focus on the input triggers :focus-visible + label styling, and the file list is still accessible. This is standard progressive enhancement — the input functions without the label styling if CSS fails.

Color Picker

<input type="color">
Browser Native
Pseudo-elements
::-webkit-color-swatch-wrapper
  /* Wrapper padding */
::-webkit-color-swatch
  /* The color swatch itself */

/* Remove border, control border-radius */
/* of the swatch via these pseudo-elements */
VB Styled
appearance: none;
padding: 3px; /* inset gap */
border: 1px solid var(--input-border);
/* Swatch border-radius matches token */
✓ Include (light touch) Recommendation
Minimal intervention needed. appearance:none removes OS chrome; the swatch pseudo-elements let VB normalize the border and radius. The OS color picker popup is unaffected — that's correct (it's native, accessible, and not VB's concern). Useful for the ThemeManager UI where color inputs edit token values.
