User Journey

Visualise multi-phase user journeys with emotion curves and phase breakdowns

← Web Components R&D

⚠ Requires a local server to load ES modules — run npx serve . or python3 -m http.server from the repo root.

Overview

The <user-journey> component renders a journey map card with an SVG emotion curve and a structured phase grid covering actions, thoughts, touchpoints, pain points, and opportunities. It cross-references <user-persona> and <user-story> via anchor links.

👤 user-persona → 🗺 user-journey ←→ 📋 user-story

persona-id links to a <user-persona id="…">. Each phase's storyIds array links to <user-story> elements.

User Journey

Five-phase journey for a Senior Developer evaluating an API management platform — from first Google search through steady-state adoption. Each phase captures real actions, internal monologue, touchpoints, pain points, and design opportunities.

Technical Evaluator — API Platform Evaluation
<user-journey title="API Platform Evaluation" persona="Sarah Chen" persona-id="persona-sarah-chen" summary="How a senior developer discovers, evaluates, and adopts the platform" src="/data/journeys/api-evaluation.json"> </user-journey>

Compact Mode

Use the compact boolean attribute for tighter layouts.

Support Incident Journey — 3 Phases, Compact
<user-journey title="Customer Support Journey" persona="End User" src="/data/journeys/support.json" compact> </user-journey>

JSON Schema (src attribute)

Fetch phase data from a JSON file for complex journeys.

{ "title": "API Platform Evaluation", "persona": "Sarah Chen", "personaId": "persona-sarah-chen", "summary": "How a developer discovers and adopts the platform", "phases": [ { "name": "Awareness", "emotion": "curious", "storyIds": ["PROJ-101"], "actions": ["Searches for API management solutions"], "thoughts": ["There must be a simpler way to manage our APIs"], "touchpoints": ["Google", "Hacker News", "Dev.to"], "painPoints": ["Hard to compare vendors"], "opportunities": ["Developer-focused landing page"] } ] }

Attributes

Attribute Type Description
title string Journey title displayed in the card header.
persona string Persona name shown in the header.
persona-id string ID of the <user-persona> on the page — renders as an anchor link.
summary string One-line description of the journey scope.
story-ids comma-separated IDs of related <user-story> elements — shown as chip links.
src URL Fetches all phase data from a JSON file.
compact boolean Reduced padding and curve height for tighter layouts.
