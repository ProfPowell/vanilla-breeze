04

Progressive Enhancement

The same component shown at each CSS capability tier. Content works at every level. Visual complexity layers on top — exactly as a supergraphic is applied to an existing building.

Tier 1 — HTML only

Vanilla Breeze Component

Content is structured, readable, accessible. No visual enhancement. The room before the painter arrives.

Tier 2 — + CSS geometry

Vanilla Breeze Component

CSS adds the supergraphic circle. It's cropped at the container edge — the geometry exists but doesn't intrude on content.

Tier 3 — + mix-blend-mode overlap

Vanilla Breeze Component

Two overlapping circles with mix-blend-mode: multiply produce a secondary colour at the intersection. Red + Blue = Violet. The primary palette generates its own secondaries.

Tier 4 — + SVG supergraphic layer

Vanilla Breeze Component

SVG layer adds precise arcs, grid lines, and blended geometry. The content sits above in the stacking context. Remove the CSS and HTML — this layer disappears, content remains.

Why Supergraphics + Progressive Enhancement

Stauffacher Solomon applied supergraphics to existing buildings — the structure came first, the art was additive. Vanilla Breeze works identically: the HTML is the building. CSS is the painter. Remove the painter and the building still stands, fully functional, perfectly readable. Add the painter and the building becomes extraordinary. The overflow: hidden on a section is the wall. The arc that enters from outside is the supergraphic. Scale is the idea. The platform is the canvas.
