Release

v2.4.0 — Page Tools & Tour Components

1 Apr 2026

New page-tools hybrid toolbar and page-tour guided walkthrough components ship with full four-layer progressive enhancement.

This release introduces two major components:

  • <page-tools> — A hybrid container/toolbar aggregating page-level utilities (print, share, text reader, reading stats). Sits inside the existing layout aside. Mobile collapse uses a FAB + native popover drawer.
  • <page-tour> — Guided walkthrough with three modes: passive, active, and forced. Uses CSS Anchor Positioning API as the primary positioning mechanism.

Both components follow the four-layer progression: native HTML → class identity → data-* configuration → web component enhancement.

Enhancement

v2.3.1 — CSS Color Modernisation

15 Mar 2026

Migration from pre-computed colour tokens to color-mix() and relative colour syntax with OKLCH seed/derived architecture.

The token system now uses OKLCH colour space as the seed layer, with color-mix() and relative colour syntax generating derived tokens at runtime. This eliminates hundreds of pre-computed values while improving perceptual uniformity.

Key changes:

  • Seed tokens defined in OKLCH
  • Derived tokens via color-mix(in oklch, ...)
  • Automatic dark mode via relative colour syntax
  • Token layer swaps remain automatic; chrome layer is bundle-owned
Release

v2.3.0 — Customisable Select

1 Mar 2026

Native customisable select using @supports (appearance: base-select) — zero-risk progressive enhancement layer.

Ships the customisable <select> upgrade gated behind @supports (appearance: base-select). Browsers that support the new spec get rich custom rendering; all others get the standard native select with no degradation.

This is pure progressive enhancement — the feature detection gate means zero risk of broken experiences.

Release

v2.2.0 — Manifest System

10 Feb 2026

Four interconnected manifest specs ship: page manifest, site manifest, content negotiation, and delivery integrity via Resource Timing API.

The manifest system preserves build-time knowledge for runtime use. The SSG has complete site knowledge that was previously discarded — now it is captured in four interconnected specifications:

  • Page Manifest — Per-page asset inventory
  • Site Manifest — Site-wide resource map
  • Content Negotiation — Multi-format delivery
  • Delivery Integrity — Verification via Resource Timing API
Release

v2.1.0 — Form Validation & Bot Protection

20 Jan 2026

Progressive enhancement form validation with bot protection via data-* attributes directly on the form element.

The form system ships with two complete specs: FORM-VALIDATION-PE.md for progressive enhancement validation, and FORM-BOT-STRATEGY.md for bot protection.

Key architectural decision: the <form> is the component. Bot protection configuration lives as data-* attributes directly on the native element — not on a wrapper.
