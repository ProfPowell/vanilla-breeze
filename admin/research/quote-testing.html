Demo: HTML <q> and foreign languages

This page lets you check whether your browser’s default rendering of <q> responds to lang or not. The first section uses browser defaults. The second forces quote marks with CSS so you can compare.

1) Browser default behavior

Page language: English

He said This is a short quote.

Nested example: She replied Not today and left.

Expected in many browsers: curly English quotes.
Language: French

Il a dit Ceci est une citation courte.

Imbrication : Elle a répondu Pas aujourd’hui puis elle est partie.

Look to see whether you get French guillemets or just English-style quotes.
Language: German

Er sagte Dies ist ein kurzes Zitat.

Verschachtelt: Sie antwortete Nicht heute und ging.

Look for German low-high outer quotes in browsers that localize them.
Language: Spanish

Él dijo Esta es una cita breve.

Anidado: Ella respondió No hoy y se fue.

Some browsers may still show generic curly quotes instead of Spanish-preferred styling.
Language: Japanese

彼はこれは短い引用です。と言いました。

入れ子: 彼女は今日はだめと答えて去った。

Check whether corner brackets appear by default.
Language: Chinese

他说：这是一段简短的引文。

嵌套：她回答今天不行然后离开了。

Check whether localized CJK quotation marks appear.

2) Mixed-language quotes inside an English page

English context with French quote: Ceci est une citation française.

English context with German quote: Dies ist ein deutsches Zitat.

English context with Japanese quote: これは日本語の引用です。

This is the key test. Some browsers respond to the quote’s own lang; others behave more like the surrounding language context. W3C’s i18n material specifically notes uneven default behavior here. [oai_citation:1‡W3C GitHub](https://w3c.github.io/i18n-drafts/questions/qa-the-q-element.en?utm_source=chatgpt.com)

3) Forced quote styles with CSS for comparison

The CSS quotes property lets you explicitly control the quotation marks used by <q>. MDN documents that this property affects the automatically added quotation marks for the element. [oai_citation:2‡MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/quotes?utm_source=chatgpt.com)

4) What to look for

Test Question
Default English Does the browser add quote marks around <q> at all?
French / German / Japanese Do the glyphs change when the container or quote has a different lang?
Mixed-language section Does the quote follow its own lang or the surrounding page language?
Forced CSS section How does browser default behavior differ from explicit typographic control?
