Animated Borders

CSS @property + conic-gradient

Spinning gradient border — conic-gradient + @property --ba

--color-primary → secondary → accent
Theme Spin

Uses --color-primary, --color-secondary, and --color-accent. Swap the theme, the border follows.

Slow (6s) — subtle, ambient
Slow Spin

At 6s the motion is almost imperceptible — a quiet sign of life on a card. Elegant on dashboard widgets.

Rainbow (2.5s) — 7-stop conic
Rainbow Spin

Full spectrum. 7 hue stops. Fast. Retro and Kawaii themes would use this; professional themes wouldn't.

Pulse border — animated box-shadow outward ring

Primary pulse — selected / active
Active Pulse

Ring expands to 8px then fades. Communicates "selected" or "focused" without a hard ring.

Danger pulse — alert / error states
Danger Pulse

Red, faster (1.5s), larger spread (10px). Urgency without a modal. Use on critical inputs or system alerts.

Multi-ring pulse — concentric rings
Multi-ring

Two simultaneous shadow rings at different radii. Radar-like. Striking on dark backgrounds.

Marching ants — background-position animation, no SVG

background-image dash pattern + animation
Marching Ants

Four repeating-linear-gradient backgrounds simulate a dashed border. Animating background-position makes them march. No SVG, no JS. Great for selection states, drag targets, or clipboard operations.

Hover to reveal — clockwise border trace
Hover to trace border

Staggered transition-delay on individual border sides (top → right → bottom → left, 180ms each). The border draws clockwise. Pure CSS — no clip-path, no JS. Works on any element with a ::before / ::after available.

Hue-cycle and Breathe — simpler effects

filter: hue-rotate(360deg) cycling
Hue Cycle

One colored border + filter: hue-rotate animating to 360°. Simplest possible technique. One property, one keyframe.

::before ring that expands + fades
Breathe

A ::before ring scales from 100% → 106% while fading out. Like a sonar ping. Good for "standby" or "recording" states.

Scrollbar Theming

scrollbar-color · scrollbar-width · Baseline ✓

scrollbar-color and scrollbar-width are now Baseline: widely available (Chrome, Firefox, Safari 18+). Two CSS properties, zero JavaScript, zero pseudo-elements. Themes set --scrollbar-thumb and --scrollbar-track; the root rule does the rest.

Default — thumb gray / track light gray
Default scrollbar: muted gray thumb against a noticeably lighter track. The thumb/track contrast is the most important thing to get right — they must be perceptibly different or the track disappears. Scroll this box to see both thumb and track. This is the base experience every theme inherits before overriding. More text here so scrolling is necessary to reach this sentence.
Primary — thumb indigo / track pale indigo
Indigo thumb against a pale indigo track — the track tint is key, it makes the gutter visible as a design element rather than dead space. The theme's brand hue carries all the way to the scroll chrome. Sidebar menus and persistent panels benefit most from this. More text so scrolling is required to reach the end of this box.
Vivid — thumb hot pink / track blush
Hot pink thumb on a blush track. Dramatic and unmistakable. Kawaii, Memphis, and retro themes would use this. The track tint makes the whole scrollbar feel intentional rather than incidental. scrollbar-color accepts any CSS color including OKLCH — full color system access. More text here to ensure the thumb is always visible on screen.
Minimal — thumb light gray / track near-white · thin
scrollbar-width: thin with muted colors. Maximum content area, minimum visual chrome. Nordic, Swiss, and editorial themes use this. The scrollbar is present and functional but doesn't compete with the content. It fades into the background until the user needs it. More text needed for the thumb to appear.
Hidden — scrollbar-width: none · still scrollable
No visible scrollbar — but the box still scrolls. Try it: use a trackpad, mouse wheel, or keyboard arrow keys. The content is accessible; only the UI chrome is hidden. Use for carousels, horizontal scrollers, and cards where a visible scrollbar would break the layout. More text below. Even more content here to make scrolling obvious is possible.
Auto — thumb green / track pale green · auto width
scrollbar-width: auto — the widest native option. Green thumb on a pale green track. This is the most accessible variant: the large thumb hit-target suits motor impairment users. A11y themes and medical/government applications should default to auto-width with high-contrast thumb-to-track contrast.

SVG data URI cursors — hover each zone

Crosshair cursor Circle + 4 lines. SVG data URI.

Purple ring + lines. Hotspot at 16,16. Falls back to crosshair.

Dot cursor Solid orange dot with white center.

16px dot, hotspot centered. Used in precision design tools and data viz themes.

Branded arrow Custom arrow in --color-primary.

Replaces the OS arrow with a theme-colored one. Full brand control at 24px. Hotspot at 5,3.

Large a11y cursor 48px ring — high visibility.

Accessibility-first cursor for motor impairment or large-display themes. Green, 48px, centered hotspot.

Implementation note: SVG cursors use inline data URIs — no separate files, no network requests. The SVG is URL-encoded, color values are hex (not OKLCH — SVG fill doesn't accept OKLCH in cursor URLs). The cursor property accepts a fallback: url("…") hotspot-x hotspot-y, fallback-keyword. Custom cursors work in all modern browsers. Max recommended size: 128×128px; 24–48px is practical.

Theme token pattern: Themes set --cursor-custom-default and --cursor-custom-pointer. The root rules apply them to :root and interactive elements respectively. JS cursor effects are opt-in via --spotlight-enabled: 1.

Move cursor — spotlight follows via CSS custom properties on the element

JS reads mousemove coordinates, converts to percentages, and sets --spotlight-x / --spotlight-y on the element. CSS renders a radial-gradient using those properties in the ::before. The browser handles smooth compositing — no canvas, no requestAnimationFrame loop needed. Dramatic on dark themes. Pair with --spotlight-size and --spotlight-color.

Reduced Motion

prefers-reduced-motion · data-motion-reduced

All animated border effects must disable when motion is reduced. The pattern follows the existing VB motion token approach — one media query stops everything. 

/* In the tokens layer — same pattern as --duration-* */
@media (prefers-reduced-motion: reduce) {
  :root {
    --border-anim-spin-duration:  0s;    /* stops animation */
    --border-anim-pulse-duration: 0s;
    --border-anim-march-duration: 0s;
    --border-anim-hue-duration:   0s;
    --spotlight-enabled:          0;    /* JS checks this */
  }
}
/* User toggle via ThemeManager */
:root[data-motion-reduced] {
  --border-anim-spin-duration:  0s;
  --border-anim-pulse-duration: 0s;
  --border-anim-march-duration: 0s;
  --border-anim-hue-duration:   0s;
  --spotlight-enabled:          0;
}

When animation-duration: 0s, the animation still runs but completes instantly — the element stays in its final animation-fill-mode state. For spin and hue borders this means the gradient is frozen at the start angle, which is still visually correct. For pulse borders it means no glow at all — fall back cleanly to a static border.
