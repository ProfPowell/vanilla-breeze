Navigation Patterns

This page demonstrates the navigation and layout patterns from Phase 2 of the pattern system. All patterns work together to create a complete, accessible page structure.

Introduction

The navigation patterns provide consistent structure and wayfinding across your site or application. They follow progressive enhancement principles—working with basic HTML and enhanced with CSS and optional JavaScript.

Patterns on This Page

  • skip-link — Press Tab to see it appear
  • site-header — Sticky navigation at the top
  • breadcrumbs — Location trail below the header
  • sidebar-layout — Two-column layout with navigation
  • page-shell — Overall page structure
  • site-footer — Footer with links at the bottom

Site Header

The site header provides primary navigation with logo, main links, and optional actions. It supports sticky positioning and responsive mobile menu.

Features

  • Sticky positioning with data-sticky
  • Mobile menu using popover API
  • Current page indication with aria-current
  • Size variants: compact, default, large
  • Style variants: transparent, solid, blur

Breadcrumbs

Breadcrumbs show the user's location in the site hierarchy and provide quick navigation to parent pages.

Features

  • Semantic <ol> structure
  • Separator variants: slash, chevron, arrow
  • Schema.org structured data support
  • Truncation for long paths

Skip Link

The skip link is hidden until focused, allowing keyboard users to bypass navigation and jump to main content. Press Tab on this page to see it.

Features

  • Hidden until keyboard focus
  • First focusable element on page
  • Targets main content area
  • Essential for accessibility compliance

Page Shell

The page shell provides the overall page structure with header, main content, and footer. It ensures consistent layout and proper landmark structure.

Features

  • Full viewport height layout
  • Layout variants: standard, full-width, centered
  • Proper landmark structure for accessibility
  • Skip link integration

Sidebar Layout

The sidebar layout creates a two-column structure with primary content and a sidebar for navigation, filters, or supplementary content.

Features

  • Position variants: start (left) or end (right)
  • Sticky sidebar option
  • Responsive with collapsible mobile sidebar
  • Works with documentation, e-commerce filters, TOC
