Overview

The <day-view> component renders an hour-grid schedule for a single day. It progressively enhances a semantic <ol> with <time> elements — without JavaScript, the content renders as a plain readable list. With JS, events are positioned into a visual hour grid with localized labels and keyboard navigation.

Markup Pattern

The component enhances existing HTML — you write a semantic <ol> and the component positions it.

ElementPurpose
<ol>Ordered list of events (chronological sequence)
<li>Individual event item
<time datetime="HH:MM">Event start time (machine-readable)
<time datetime="PT1H">Optional event duration (ISO 8601 format)
<data value="category">Optional metadata (meeting, deadline, personal)
<details>Optional expandable event information

Event Categories

Add a <data value="category"> element inside an event <li> to categorize it. The component reads the value and sets data-category on the <li> for CSS targeting. Built-in categories:

CategoryColor
meetingPrimary (blue/interactive)
deadlineError (red)
personalSuccess (green)

Custom categories work via CSS: day-view li[data-category="workshop"] { --event-color: purple; }

    Progressive Enhancement

    The component works in three layers:

    LayerWhat renders
    HTML onlyOrdered list with times — readable, semantic, accessible
    + CSSStyled list with time formatting and category colors
    + JSHour grid positioning, localized labels, keyboard nav, click events

    Events

    EventDetailDescription
    day-view:event-click { time, text, element } Fired when an event item is clicked.

    Keyboard

    KeyAction
    Arrow DownMove focus to next event
    Arrow UpMove focus to previous event

    Localization

    Hour labels are rendered via Intl.DateTimeFormat, matching the user's locale. Override by setting lang on the element or an ancestor.

    Accessibility

    • Semantic <ol> preserves list semantics for screen readers
    • <time datetime> provides machine-readable times
    • <details> elements are natively keyboard-accessible
    • Roving tabindex on event items for arrow key navigation
    • Hour labels are aria-hidden (visual aid, not content)
    • Focus ring visible on focused event
